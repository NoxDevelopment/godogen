# Point-and-Click Adventure Template

Classic adventure-game base on **Popochiu 2** (carenalgas). Scaffold with:

```bash
python templates/tools/scaffold.py point-and-click <target-dir> --name "Game Name"
```

Engine pin: **Godot 4.6.x** (validated on 4.6.1-stable) with Popochiu **v2.1.1**
(pinned tag + commit in the registry).

## What you get

- **Popochiu fully initialized** — the state the plugin's setup wizard normally
  produces is pre-baked into `game/`: the ten autoloads (`Globals`, `Cursor`,
  `E`, `R`, `C`, `I`, `D`, `A`, `G`, `T`), `popochiu_data.cfg` (migration
  version, GUI template, rooms/characters, PC), and the transition layer.
- **simple_click GUI** copied to `game/gui/` (left-click interact, right-click
  look, bottom bar with inventory + settings, dialog text, history/save/load
  popups) — all components are game-local copies you can restyle freely.
- **One room** (`game/rooms/studio/`) with:
  - a **walkable area** (`Floor`) covering the floor strip — click anywhere on
    it and the PC pathfinds there;
  - a **clickable prop** (`Poster`) with working interact/look handlers
    (walk-to + face + `say(...)` lines) — the full interaction loop, live;
  - a flat-color blockout backdrop (wall / floor / poster art placeholders).
- **Player character** `Hero` (`game/characters/hero/`), registered as the PC,
  spawned onto the walkable area at room entry.
- **NoxDev template ABI**: `Master`/`Music`/`SFX` buses
  (`default_bus_layout.tres`), Popochiu's input actions (`popochiu-interact`,
  `popochiu-look`, `popochiu-skip`) plus `pause` baked into `[input]`,
  320x180 pixel-art render settings (window override 1280x720).

## How the pieces fit

Popochiu's runtime boots from the `E` autoload; the **main scene is the room
scene** (`room_studio.tscn`). Rooms/characters/props are managed through the
**Popochiu dock** in the editor (bottom-left) — create new ones there, never by
hand-copying folders, so `popochiu_data.cfg` and the `R`/`C`/`I`/`D` autoload
facades stay in sync. Naming note: don't name a character "Player" — it is a
reserved alias in Popochiu's character API (this template's PC is `Hero`,
accessed as `C.Hero` or `C.player`).

## How to extend

1. **Rooms**: Popochiu dock → Rooms → Create room; set one as main to change
   the start room. Navigate with `R.goto_room("RoomName")` from any handler.
2. **Props/hotspots**: create via the dock with the room scene open; implement
   `_on_click` / `_on_right_click` or `on_<command>` functions (see
   `prop_poster.gd` for the working pattern).
3. **Dialogs and inventory**: dock → Dialogs / Inventory items; branching
   dialogs use `D.SomeDialog.start()`; items land in the `I` facade.
4. **Art**: replace the blockout `Backdrop` polygons with a 320x180 background
   (a `bg` prop at z-index -1 is the Popochiu convention), give `Hero`
   idle/walk/talk frames in `character_hero.tscn`, swap the poster's sprite and
   re-fit its interaction polygon. See `assetPlanHints` in the registry.
5. **GUI reskin**: everything under `game/gui/` is yours; pair with the
   godogen `ui-theme` skill for a palette-driven theme.
6. **Settings**: godotsmith's `settings_system` drop-in works (buses match);
   Popochiu's own save/load popups already handle game saves.

## Validation status

`status: "validated"` — scaffolded (bootstrap import + deferred plugin enable),
`--headless --import` and a 120-frame headless boot both produce **zero script
errors** on Godot 4.6.1; runtime probe confirms the PC spawns on the walkable
area at (160,160) with the prop clickable. The single remaining log line
(`1 RID allocations of type 'NavRegion2D' were leaked at exit`) is engine
shutdown accounting from the walkable area's NavigationRegion2D (upstream
Popochiu behavior, exit code 0, not a script error).

Scripts referencing Popochiu autoloads can't be run through
`--check-only --script` (autoloads don't exist there); the headless boot is the
authoritative check. Autoload-free scripts (`walkable_area_floor.gd`,
`popochiu_globals.gd`, state scripts) pass `--check-only`.
