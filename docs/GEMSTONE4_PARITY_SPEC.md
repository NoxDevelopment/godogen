# GemStone IV Parity Spec — the text-MUD/MMORPG build bible

> Reference target for the `text-rpg-mud` flagship (Godot client + Evennia BSD server). Compiled from web research (Play.net, GSWiki/Elanthipedia, fan client docs) by workflow wf_91a31fc8. Parity = GemStone IV's UI/UX, systems depth, world + lore, and living-world autonomous agents, built with our own assets/apps. Nothing is 'done' until Jesus vets it.

**Architecture keystone:** GS4's client is essentially a *tag renderer* over one server-pushed markup stream. We adopt that: define the tag/style protocol as our wire format, make the Evennia server the emitter, and the Godot client a stateless renderer (docked panels + point-and-click verb menus + themeable presets). Our WORLDBUILDER feeds rooms/entities/lore, the COMPANION library drives autonomous NPC personalities (consume-only), LLM generation writes descriptions/dialogue, and the RULESET builder defines the numbers the vitals bars render.

**Status:** Evennia server LIVE on Postgres (telnet 4000 / websocket 4002 / webclient 4001); living-world mechanic PROVEN (autonomous world-clock + wandering agent, verified uncached). This spec drives the depth build from here.

## GemStone IV client UI/UX parity spec — front-end layout, text-stream tag taxonomy, and Godot panel plan (Wizard / StormFront-Wrayth / Genie + Lich)

GemStone IV is played through three front-ends that all consume the SAME server-pushed markup stream, so the client is essentially an XML-tag renderer. The server emits a hybrid of plain game text and inline control tags; the front-end routes tagged spans to panels (vitals bars, hands, compass, injuries figure, active-spell timers, side windows) and applies style presets (roomName, speech, whisper, thought, monsterbold, familiar, voln) to color the main scroll. Simutronics' WIZARD (classic, 2000) is a floating multi-window layout with a compass rose, hands line, H/M/S body graphic, roundtime clock, mana-pulse timer and 5 quick buttons. STORMFRONT/WRAYTH (2003, the rich GUI) formalized this into a docked three-pane layout driven by ~60 XML tags (streamWindow/pushStream, dialogData/progressBar, compass/dir, indicator, left/right, spell, a/d clickable links) with point-and-click context menus, an injuries body diagram, and an Active Spells dialog with per-spell countdown meters. GENIE is the fan client (user-defined #window panels, highlights, auto-map, runs Wizard/Wrayth scripts); LICH is a Ruby proxy that sits between server and any FE. To hit GS4 parity in Godot we replicate the tag taxonomy as our server protocol and build a docked panel client that renders it: a central RichTextLabel scroll + a status strip (4 vitals bars, RT bar, stance/encumbrance/mind-state, posture icons), hands line, compass rose, and dockable side panels (room, inventory, active spells, injuries figure, thoughts/ESP, logons, death, group, familiar). Every span in the main stream is a semantic style class and every interactive noun is a clickable link carrying an object id + verb menu.

### Findings

**Three official/fan front-ends — what each is and which to model**  
WIZARD (WizardFE, ~2000): Simutronics' classic Windows client. Floating/resizable multi-window interface; game text can be routed to separate windows (Thoughts, Death, Familiar, Arrivals, Wounds, Pictures). Shows a Hands line (left/right + prepared spell), a compass/Direction display (compass rose, yellow arrowheads = available exits), a Health display (Health/Mana/Spirit bars + small body graphic), a Status display (field-exp saturation, defensive stance, active spell effects), a Roundtime display (current + spell RT), a 2-minute Mana Pulse timer, a bottom command buffer, and 5 quick-action buttons (equip/unequip weapon+shield). Macro tokens: \r (Enter), @ (cursor), \? (prompt for input), \p (pause). STORMFRONT / WRAYTH (2003, renamed Wrayth ~2020): the rich point-and-click GUI, the primary parity target. Docked panes, click any text to get a verb context-menu, XML-driven dialog panels, skins (Original Storm, StealthFront, NishiFront, Brute), per-window logging (floppy icon), toggle all panels with the {panels} command. GENIE (Genie3, genieclient.com): third-party Windows client for all Simutronics games; user-creatable windows via #window, highlights/aliases/variables/scripting, auto-mapping, runs Wizard/Wrayth scripts. LICH: Ruby scripting engine that runs as a PROXY between the server and whatever FE you use (Wizard/Wrayth/Genie) — not a renderer, it injects/reads the same tag stream. Also: Profanity (terminal FE), Avalon (Mac), Web Client, Java client all consume the same protocol.
  
_Build note:_ Model the Godot client on StormFront/Wrayth (docked panes + point-and-click + tag-driven panels). Keep Wizard's floating-window option as a 'classic mode'. Treat the server->client markup as OUR wire protocol so scripting layers (our Lich-equivalent) can proxy it.

**The whole client is a tag renderer — protocol is the architecture**  
All FEs consume one server-pushed stream that interleaves plain game text with inline SGML/XML-ish control tags. The protocol page documents 60+ elements. Categories: system (mode, playerID, settingsInfo, app, endSetup), stream mgmt (streamWindow, pushStream/popStream, clearStream, stream, exposeStream), dialog/UI (openDialog/dialogData/closeDialog/exposeDialog, progressBar, cmdButton, label, image, closeButton), character state (prompt, roundTime, castTime, indicator, spell, left, right), navigation (a, d, compass, dir, nav), components/containers (compDef, component, container, clearContainer, exposeContainer, deleteContainer, inv), text formatting (pushBold/popBold, b, output, style, preset), settings/control (flag, menu, mi, monopolize, switchQuickBar, updateverbs), plus skin, dropDownBox, upDownEditBox, menuLink, radio, link, sep, pushInputState/popInputState, cmdlist, cmdtimestamp. mode id values: GAME/LOGIN/CMGR.
  
_Build note:_ Define this tag set as our canonical text-protocol schema first; the Godot client is a stateless renderer over it and the Evennia server is the emitter. This is the single most load-bearing decision for GS4 parity.

**Main text stream — styling / preset taxonomy**  
The central scroll interleaves everything not routed elsewhere. Styling is driven by <style id=...> and <preset id=...> spans plus bold (<pushBold>/<popBold> or <b>) and font switches (<output class='mono'> for fixed-width tables, empty class to return to proportional). Named presets/style ids seen: roomName (room title line), roomDesc, speech (player says/asks/exclaims), whisper, thought / thoughts (ESP), monsterbold (creature names — the bold-highlighted nouns you can attack/target), familiar (familiar-relayed text), voln (society channel), plus the highlight system (preset highlights + user custom highlights + a color palette accessed by right-click > Settings). Each preset maps to a user-editable fg/bg color and font. monsterbold is special: creature/notable nouns are wrapped so they render bold AND are clickable targets.
  
_Build note:_ In Godot use a RichTextLabel with BBCode; map each preset id to a theme color/style. Ship sensible defaults (room title = tan/gold, speech = pale green, whisper = dim, thought/ESP = cyan/blue, monsterbold = bright bold, familiar = purple, self-actions = white) and make every preset a user-remappable palette entry. Support a monospace toggle for aligned tables (EXP, INVENTORY, SKILL).

**Clickable text + point-and-click verb menus**  
Interactivity is inline: <a exist='OBJECTID' noun='NOUN'>display text</a> marks any object (item, creature, player, exit) as clickable and carries a persistent object id + its noun; <d cmd='COMMAND'>text</d> is a direct command link that runs COMMAND on click. In StormFront, left/right-clicking a highlighted noun opens a context menu of applicable verbs (LOOK, GET, ATTACK, ANALYZE, etc.); the verb list is refreshed by <updateverbs>/<menu>/<mi coord=...>. Links also carry char/game/coord attributes for cross-game and coordinate menus. <link URL=...> and launchURL open external browser pages.
  
_Build note:_ Emit <a exist noun> for every referenceable entity from the server so the Godot client can make nouns clickable. Implement a right-click radial/context menu that requests the verb list for that object id, then sends the chosen verb+noun as a command. This is core GS4 UX and where our LLM/worldbuilder object ids plug in.

**Vitals / status bars (Health, Mana, Spirit, Stamina) + derived meters**  
Driven by <progressBar id=... value=0-100 text='...' top left height width align tooltip>. Documented progressBar ids: health, health2 (secondary/overlay), mana, spirit, stamina, pbarStance (stance meter), encumlevel (encumbrance), mindState (experience saturation/mental acuity), nextLvlPB (progress to next level). Each bar shows a fill value 0-100 plus a text overlay (e.g. 'health 100/100', 'mana 84/120'). Colors are user-customizable (defaults roughly: health red/green, mana blue, spirit yellow/gold, stamina green). Health/Mana/Spirit/Stamina come from the HEALTH/MANA/etc. verbs and update live.
  
_Build note:_ Build a StatusStrip with four labeled ProgressBar widgets (health, mana, spirit, stamina) each with a value + text overlay, driven directly by progressBar tags keyed on id. Add pbarStance, encumlevel, mindState, nextLvlPB as additional bars. Keep colors themeable per bar (StormFront lets users recolor each).

**Roundtime (RT) and cast time bars**  
<roundTime value='SECONDS'> sets an action lockout; <castTime value='SECONDS'> is the spell-prep timer (shown separately in Wizard's Roundtime display which shows current RT AND spell RT). The server also sends <prompt time='UNIXTIME'> before each command prompt, so the client knows current server time and can count RT down accurately. Classic display is a shrinking colored bar / numeric countdown; combat text prints 'Roundtime: 5 sec.' and the bar drains over those seconds. During RT typed commands queue or are rejected.
  
_Build note:_ Render an RT bar that fills to roundTime.value and drains to zero using prompt.time as the server clock reference (compute remaining = (rt_start + value) - server_now). Show a separate cast-time bar for castTime. Gray/queue the input line while RT>0 as an option.

**Stance, encumbrance, and posture/status indicator icons**  
Defensive stance shown via pbarStance progressBar and in the Status display (offensive<->defensive). Encumbrance via encumlevel progressBar (none/light/... /overloaded affecting RT and dodge). Posture & status conditions are boolean icons via <indicator id=... visible='y|n'>. Documented indicator ids: IconKNEELING, IconPRONE, IconSITTING, IconSTANDING, IconSTUNNED, IconHIDDEN, IconINVISIBLE, IconDEAD, IconWEBBED, IconJOINED. Only one posture (standing/sitting/kneeling/prone) is active; stunned/hidden/invisible/dead/webbed/joined layer on top.
  
_Build note:_ Add a row of toggleable status icons bound to indicator ids (posture group = mutually exclusive; condition group = independent flags). Bind stance + encumbrance to their progressBars. joined = grouped/following indicator.

**Hands display (left / right / prepared spell)**  
<left exist='OBJECTID' noun='NOUN'>item text</left> and <right ...>item text</right> report what is held in each hand (empty when exist absent). <spell>SPELL NAME</spell> reports the currently prepared/readied spell (e.g. 'Minor Shock', or 'None'). Classic Wizard shows this as a three-cell line: Left Hand: ... | Right Hand: ... | Spell: ....
  
_Build note:_ Build a HandsPanel with three fields (Left Hand, Right Hand, Prepared Spell) bound to left/right/spell tags; items are clickable (carry exist/noun) to open the same verb menu. This is glanceable combat state — keep it near the input line.

**Compass rose + directional movement + obvious exits**  
<compass> contains <dir value='...'> child elements for each currently available exit; value set = n, s, e, w, ne, nw, se, sw, up, down, out. The FE renders a compass rose (8 compass points + up/down/out) and highlights only the directions present (Wizard: yellow arrowheads on available exits). Clicking a lit direction sends the movement command. Separately, the room text prints a verbatim 'Obvious exits:' (or 'Obvious paths:') line, e.g. 'Obvious exits: east, northwest'. <nav rm='ROOMID'> announces room changes for auto-mappers.
  
_Build note:_ Build a CompassRose widget (8 diagonal/cardinal buttons + Up/Down/Out) driven by the compass/dir set; lit buttons are clickable and send movement. Also parse/emit the textual 'Obvious exits:' line in the room stream. Emit nav rm=... on every room transition for our auto-map.

**Room display format + room components/window**  
Room prints as: a bracketed title line [Area, Room Name] styled roomName; then the description paragraph (roomDesc); then a 'You also see ...' objects line; then 'Also here:' listing players/creatures/bodies present (e.g. 'Also here: Yaristen, Loralai, the body of Lord Reminion who is lying down'); then the 'Obvious exits:'/'Obvious paths:' line. StormFront also mirrors current room into a dedicated Room window via component tags: <component id='room desc'>, <component id='room objs'> (the 'you also see' objects), <component id='room players'> (the also-here list), <component id='room exits'>, and <component id='sprite'> (room art id). Components are re-pushed on any change so the Room window stays live.
  
_Build note:_ Render the room in the main scroll AND mirror into a dockable Room panel using the 4 components (desc / objs / players / exits). Title line clickable-styled, objects and players carry <a exist noun> for verb menus. sprite id -> optional room illustration (feed from worldbuilder art). This is where the worldbuilder's room/entity data maps 1:1.

**Active Spells window with per-spell countdown timers**  
The Active Spells dialog (StormFront) graphically lists every active spell/effect with a timer and a meter showing remaining duration; users right-click an individual spell to recolor its bar. Populated by dialogData/progressBar tags in a dedicated dialog. The SPELL ACTIVE verb lists actual durations sorted: miscellaneous effects first (cooldowns, combat-maneuver cooldowns), then positive effects, then negative effects, then by spell number. Combat maneuver cooldowns also appear here.
  
_Build note:_ Build an ActiveSpells dock: one row per effect = name + draining ProgressBar (duration) + numeric time left, fed by dialogData targeting an 'activespells' dialog with a progressBar per effect. Sort misc/cooldowns, buffs, debuffs, by id. Per-row recolor. Our ruleset builder defines spell durations; the server emits the countdowns.

**Injuries / wounds / scars — body-part diagram**  
14 wound locations: head, neck, chest, abdomen, back, right eye, left eye, right arm, left arm, right hand, left hand, right leg, left leg, nervous system (nerves). Each location can carry a WOUND and/or a SCAR at severity rank 1/2/3. Rank1 = minor (combat-maneuver penalties only; rank1 scars have NO penalty); Rank2 = moderate (usually bleeds, substantial penalties, can restrict actions); Rank3 = severe (restricts activities per location — e.g. rank3 right hand can't hold/swing a weapon or cast most spells; head wounds block searching/spellcasting). Scars = partially-healed wounds (herbs leave scars; empaths heal without scarring). Wizard has a graphical Wounds window (body representation); StormFront shows an injuries figure/panel where each body part is color-coded by worst wound/scar severity. HEALTH verb prints wounds+scars, max/current HP, spirit, stamina (+recovery), phys/magic damage reduction, bleeding location+rate/round, poisons, diseases, overexertion.
  
_Build note:_ Build an Injuries panel: a front/back humanoid figure with 13 hit-region overlays + a nerves indicator, each tinted by worst wound(red)/scar(blue?) at rank 1/2/3 (light->dark). Drive it via indicator/dialogData tags per location (e.g. injury id + severity). Also parse the HEALTH text block. Our ruleset builder owns crit->wound tables; server emits per-location state.

**Experience / mind-state ('as clear as a bell' ladder)**  
Field-experience pool fills as you act; the mental-acuity ladder (from the EXP verb, empty->full, as fraction of capacity): 'clear as a bell' (0.0), 'fresh and clear' (>0.0), 'clear' (0.25), 'muddled' (0.5), 'becoming numbed' (0.62), 'numbed' (0.75), 'must rest' (0.90), 'completely saturated' (>1.0, imperative to rest). Pool capacity ~= 800 + LOG + DIS + Wisdom bonus + subscription bonus. Experience absorbs from the field pool into permanent XP via periodic 'pulses' (on-node ~25/pulse, in-town ~22, elsewhere ~19; modified by Logic, pool fullness, group). mindState progressBar reflects saturation; nextLvlPB reflects progress toward next level.
  
_Build note:_ Bind mindState progressBar to the saturation fraction and label it with the current ladder string; bind nextLvlPB to level progress. Show a mana-pulse / absorption-pulse timer (Wizard had a 2-min mana clock). Our ruleset builder can redefine the ladder labels/thresholds per game.

**Side / stream windows and how output is routed to them**  
<streamWindow id=... title=... subtitle=... location='left|right|center' target=... resident=... save=... scroll=... timestamp=...> defines a dockable window; <pushStream id='X'>...text...<popStream/> routes the enclosed text to window X instead of main; clearStream/exposeStream manage them. Documented stream ids: main, room, inv (inventory), Spells, familiar, thoughts (ESP/LNet), logons (arrivals/logins), death (death notices), speech, ambients, announcements, bounty, society, loot, charprofile, charsheet. StormFront's canonical panes: center = main story; a left long box = Spell list; a right long box = Backpack/inventory contents; top-left = experience bar; a Combat window with an adjacent Injuries/scars/health window. Thoughts window: with it open, ALL ESP messages go there instead of the game window. ESP system (THINK/CHAT/ESP TUNE, IC+OOC channels, 2020 overhaul) + fan LNet chat both feed the thoughts stream; scripts can reroute private channels to the familiar window. Containers use <container id=... title=... location=...> + <inv id=...> lines + clearContainer/exposeContainer/deleteContainer for open backpacks/boxes.
  
_Build note:_ Build a docking system where each stream id = a TabbedPanel/dock target; pushStream/popStream route spans. Ship default docks: Room, Inventory, Active Spells, Injuries, Thoughts/ESP, Logons, Death, Group, Familiar, Bounty/Society, Loot. Container tags open live per-container content panels (nested backpack view). Let users add custom windows (Genie #window parity).

**Command input, history, quick bars, and input state**  
Bottom command line sends raw commands; <prompt time=...> marks each prompt (also the RT clock source). Command history recall (up/down). Quick-action buttons: Wizard's 5 buttons; StormFront <cmdButton id value cmd echo tooltip>, <switchQuickBar id>, and macro/hotkey bars are server- or user-defined. <pushInputState state=...>/<popInputState> switch input mode (e.g. password masking at login, menu selection, CMGR character-manager). <monopolize id='main'> temporarily blocks other streams (modal prompts). <cmdlist>/<updateverbs default=...> supply the default verb menus.
  
_Build note:_ LineEdit input with history stack + tab-complete on visible nouns; a configurable quick-button/macro bar bound to cmdButton tags; honor pushInputState for password masking and modal menus; support a login/character-manager mode (mode id=LOGIN/CMGR). Support macro tokens (Enter, cursor pos, prompt-for-input, pause) for Wizard-parity scripting.

**Concrete Godot client layout (widget tree)**  
Recommended docked layout mirroring StormFront: TOP BAR = experience/level (nextLvlPB) + mind-state label + game/char title (app tag). LEFT COLUMN (VBox, dockable) = Active Spells list (top), then a general stream tab group. CENTER (largest) = main story RichTextLabel scroll (BBCode). RIGHT COLUMN = Room panel (desc/objs/players/exits) over Inventory/backpack container view. BOTTOM STATUS STRIP (above input) = [4 vitals bars H/M/Sp/St] + [RT bar + cast bar] + [stance + encumbrance + mind-state bars] + [posture/status icon row] + [Hands line L/R/Spell] + [Compass rose]. BOTTOM = command LineEdit + quick-button bar. FLOATING/TABBED (toggle with a {panels}-style hotkey) = Thoughts/ESP, Logons, Death, Familiar, Group, Bounty, Society, Loot, character sheet/profile. All panels dockable/detachable/resizable with saved layouts + skins/themes.
  
_Build note:_ Implement with Godot Control nodes: root = a docking container (or use a plugin like Godot's built-in split containers / a docking addon). Central = RichTextLabel (scroll_following, BBCode on, monospace toggle). Bars = ProgressBar keyed by tag id in a dictionary so the renderer just does bars[id].value = v. Persist layout + palette to user settings (StormFront exports/imports XML layout — offer JSON layout export/import).

### How our apps feed it

- WORLDBUILDER -> feeds the room/entity data behind the tag stream: room titles (roomName), descriptions (component 'room desc'), objects ('room objs'), exits (compass/dir + component 'room exits'), room art ids (sprite), and the persistent object ids used in <a exist=...> clickable nouns. Timelines/lore drive ambient/announcement streams.
- COMPANION AI library (consume-only) -> drives autonomous NPCs whose speech (preset speech), whispers, ESP thoughts (thoughts stream) and emotes flow through the same styled main stream and 'Also here:' list; their personality/psychology decides verb choices behind clickable NPC menus.
- LLM GENERATION -> produces room descriptions (roomDesc), NPC dialogue lines (speech/whisper), thought/ESP chatter (thoughts), and combat/ambient flavor, all emitted already wrapped in the correct style presets + clickable <a exist noun> spans.
- RULESET BUILDER -> defines the numbers the bars render: HP/mana/spirit/stamina maxima (progressBar values), roundTime/castTime durations, the wound/scar location+rank crit tables (injuries figure), spell durations (Active Spells countdowns), the experience mind-state ladder labels/thresholds (mindState/nextLvlPB), stance and encumbrance effects.
- GODOT CLIENT (this build) -> a stateless tag renderer over our text-protocol: docked panels + point-and-click verb menus + themeable style presets + saved layouts; classic 'Wizard mode' (floating windows) and rich 'StormFront mode' (docked) as two presets.
- LICH-EQUIVALENT scripting proxy (future) -> our automation layer sits between Evennia server and the Godot client reading/injecting the same tag stream, so scripts work without touching the renderer.

## GemStone IV — World, Lore & Living-World / Autonomous-Agent parity spec (for the text-MUD/MMORPG template: Godot client + Evennia server)

GemStone IV (Simutronics, set in the world Elanthia / continent Elanith, in-fiction "Modern Era" year = real year + 20, so mid-2026 ≈ 5126 M.E.) is a 35-year-old text MMORPG whose depth comes from four pillars we can mirror: (1) a large hand-authored WORLD of interconnected towns each with its own culture and a ring of level-tiered hunting areas; (2) deep LORE — a 30+ member Arkati pantheon split into Liabo (light), Lornon (dark, incl. the destruction "trinity" Luukos/Sheru/Marlu), and Neutral, plus 13 playable races, the eight Elven Great Houses, and a 100,000-year timeline (Drakes → Ur-Daemon War → Ages of Darkness/Chaos → Elven Empire → Turamzzyrian Empire → Modern Era storylines); (3) tick-driven CREATURE ECOLOGY (spawn/despawn tied to player presence, population caps, wandering, aggro, AS/CS/DS/TD combat, maneuvers, treasure, skinning, respawn); and (4) a LIVING WORLD driven by GameMaster storylines/festivals, a strong player-RP culture, a 1:1 real-time Elanthian calendar whose months/days are named after the gods, regional weather, a silver economy with keeper-run shops (furrier/gemshop/pawnshop/bank) and player-run shops, a dynamic Adventurer's Guild BOUNTY system (10 procedurally-assigned task types), and three anti-undead/anti-Grimswarm Societies. We can reach parity — and exceed GS4 because GS4's "life" is largely GM-driven and manual — by simulating autonomous agents with Evennia tick loops (world clock, spawners, NPC daily schedules, faction pressure) enriched by our Companion-AI personalities (consume-only) and LLM-generated flavor. This spec ends with a concrete first region: a Wehnimer's-Landing-style starter harbor town (~30 rooms, keeper-run shops, scheduled townsfolk agents) plus a tiered newbie hunting area (sewer rats → environs kobolds/goblins → old-road hobgoblins/gnolls).

### Findings

**Continental geography of Elanith (the parity world map)**  
Elanthia is the world; Elanith is the main known continent (part of the greater landmass Jontara, north of the continent Finnia). Regions/towns: DARKSTONE BAY region — Wehnimer's Landing, the oldest/original human frontier hub. NORTHWATCH (far north) — Icemule Trace (halfling town) and Pinefar Trading Post. ELVEN NATIONS (east, across the DragonSpine mountains via the Locksmehr Trail) — Ta'Illistim (scholar-city capital), Ta'Vaalor (fortified warrior city-state), Cysaegir, and the hamlet Sylvarraend. TURAMZZYRIAN EMPIRE lands (south/southwest) — Solhaven/Vornavis (free port), River's Rest (bayou town). ISLES — Kharam-Dzu, the dwarven mining town on Teras Isle; Kraken's Fall. DWARVEN — Zul Logoth, an underground city on the Locksmehr Trail through the DragonSpine. Key features: DragonSpine mountain range (major east-west barrier), Darkstone Bay, Lysierian Hills, Dragonsclaw Mountains. **Creature (mob) mechanics — the ecology engine**  
SPAWN: creatures spawn when adventurers are detected in a hunting area and become active. WANDER: active creatures roam the area looking for targets; rarely they wander into adjacent zones or even into a town. DESPAWN: when adventurers leave, active creatures have a chance to despawn, and any items they carried are removed from the world. COMBAT: each creature has level, family, body type, undead flag; attacks use an Attack Strength (AS) that varies by stance and health; some cast warding spells at a fixed Casting Strength (CS); many use combat/creature MANEUVERS (special moves). Defenses: Defensive Strength (DS), Unarmed Defense Factor (UDF), Target Defense (TD). An ACTION TIMER governs how often a creature acts (≥12s = slow, ≤6s = fast; haste halves it). LOOT: on death, SEARCH/loot yields treasure (silvers, gems, boxes, items); some creatures are SKINNABLE (pelt value depends on First Aid + Survival skill, weapon used, kneeling, dexterity, and whether the skin area was damaged). Aggressive creatures attack on sight; others are passive until provoked.
  
_Build note:_ Implement as Evennia scripts: (a) a per-zone Spawner (ticker) that keeps population between min/cap only while players are present, tracks respawn timers, and cleans up loot on despawn; (b) a Creature typeclass with AS/CS/DS/TD/action-timer stats + a maneuver list + a loot table + optional skin entry; (c) a Wander behavior (random-walk within zone bounds, small leak chance to adjacent room); (d) Aggro on player entry for aggressive families. Define all stat blocks and loot/skin tables in the RULESET BUILDER.

**SYNTHESIS — how WE simulate autonomous agent lives in text (tick loops + Companion AI + LLM)**  
Layered simulation on the Evennia server: (A) WorldClock ticker derives Elanthian date/time (1:1 real time) → drives day/night, seasons, holidays, and every schedule. (B) Zone Spawners run creature ecology (spawn only near players, population caps, respawn timers, wander, aggro, loot/skin, despawn-cleanup) — no LLM in the hot loop, pure rules for determinism/cost. (C) NPC Schedule state-machines: each townsfolk agent has a daily routine keyed to the clock (baker bakes at dawn, tavern fills at night, guards patrol a beat, a wandering merchant circuits towns weekly, children play in the square by day). (D) Companion-AI binding (consume-only): named NPCs each instantiate a Companion personality (FFM/HEXACO traits, cultural origin matched to race, emotional state, episodic memory of the player) that selects/colors dialogue and reactions — so the furrier remembers you, the crier's mood shifts with events. (E) LLM flavor layer (cached/budgeted): authoring-time room descriptions, on-demand ambient one-liners, crier announcements tied to real world-state, weather prose, bounty/quest flavor, and event beats — always constrained by lore-graph context to stay canon. (F) Faction pressure meters (Luukos undead, Grimswarm, etc.) tick toward thresholds that fire EventDirector incursions/festivals, giving emergent, GM-free 'sagas.' Net effect: a world that ages, works, worships, trades, and fights on a clock — matching GS4's hand-run liveliness but automated and always-on.

## GemStone IV systems depth — parity spec + prioritized build checklist for the Godot/Evennia text-MMORPG template

GemStone IV (Simutronics, set in Elanthia) is a d100 open-ended, roundtime-gated MUD whose depth comes from a small set of tightly-interlocking, fully numeric systems — all of which are natural fits for our data-driven RULESET builder + computed engines, an authoritative Evennia (BSD) server, a Godot text client, and a WORLDBUILDER/companion-AI/LLM content pipeline for the living world. The load-bearing loop is: 10 stats -> stat bonuses -> AS/DS/CS + training points -> skills -> combat/magic resolution (AS+AvD+d100-DS>100; damage=(endroll-100)xDF; critical rank from armor divisor) -> wounds/HP/bleed -> healing -> experience into a field pool -> mind-state-gated absorption -> levels/training. Everything below is a table + a pure function, which is exactly what a ruleset builder authors and a computed engine evaluates; the server enforces roundtime + pulses + persistence, the worldbuilder authors Elanthia (rooms, shops, nodes, society HQs, spawns), companion-AI gives NPCs inner lives, and LLM generates room/critical/dialogue text. Primary sources (all authoritative): GSWiki Combat_system, Statistic, Character_creation, Skill, Training_point, Magic, Spell_circle, Critical, Wound, Experience, Spellburst, Combat_Maneuvers, Enchant, Silver_coin/Bank/Locker/Box/Trap, Order_of_Voln/Guardians_of_Sunfist/Society (gswiki.play.net). Numbers below are quoted from those pages. Build the vertical slice in the checklist order; treat every table as ruleset data so a new authored world can reskin GS4 mechanics without code changes.

### Findings

**Professions (10 + Savant) and the square/semi/pure tier system**  
Ten playable professions grouped by spellcasting tier. PURE (3 spell circles, physically weak, cannot wear heavy armor, expected to train Spell Research ~2x/level, and each gets Alchemy via guild skills): Wizard, Cleric, Empath, Sorcerer. SEMI (1 minor + 1 professional/base circle, hunt primarily with a weapon supported by magic, wear >= hard leather, train weapons ~2x AND Spell Research 2x): Bard, Ranger, Paladin. SQUARE (up to 2 minor circles, combat-focused, heavy armor, little/no spell research): Warrior, Rogue, Monk. Hybrids: Sorcerer = Spiritual+Elemental; Empath = Spiritual+Mental. Monk = unarmed combat enhanced by mental magic; Rogue = stealth/ambush/locksmithing; Warrior = heavy-armor frontline. Savant is announced as a future (mental-hybrid) profession. Distinct concept: each profession has a GUILD (profession-only skills, e.g. Rogue lockpick mastery, Bard Enchante, pures' Alchemy) which is separate from cross-profession SOCIETIES (Voln/Sunfist/Light).
  
_Build note:_ Ruleset builder authors a Profession table {name, tier(pure/semi/square), primeStats[2], accessibleCircles[], skillMaxPerLevel matrix (0x/1x/2x/3x per skill), guildSkills[]}. tier drives spell-research max and armor expectations; computed engines never hardcode a class.

**Races (13) with per-stat max bonuses, growth interval, decay timer**  
Thirteen playable races: Aelotoi, Burghal Gnome, Dark Elf, Dwarf, Elf, Erithian, Forest Gnome, Giantman, Half-Elf, Half-Krolvin, Halfling, Human, Sylvankind. Each race has a maximum non-enhanced stat bonus at raw 100 that differs per stat (examples: Giantman Strength +40, Halfling Dexterity +40, Dwarf Constitution +40, Burghal Gnome Logic +35); total bonus budget across all 10 stats ranges ~255-275 per race. Each race also has a Stat Growth Interval (SGI) modifier affecting how fast stats climb on level-up, a body-decay timer used at death (Human ~14 min, Elf ~10 min), and a size that shifts hit-location distribution.
  
_Build note:_ Race table in ruleset {statMaxBonus[10], sgiModifier, decayTimerSec, size}. Worldbuilder can rename/reskin races per authored world; computed max-bonus + growth stays engine-driven.

**Ten statistics, the bonus formula, growth, and what each stat drives**  
Stats: Strength, Constitution, Dexterity, Agility (physical); Aura, Discipline (hybrid); Logic, Intuition, Wisdom, Influence (mental). Raw range 20-100. Bonus = floor((raw-50)/2) + raceModifier (raw 50 => +0 pre-race; +1 per 2 points over 50). Effects: STR->melee AS, encumbrance, stamina, block/parry DS; CON->Health Points, crit/disease resistance, encumbrance; DEX->ranged AS, spell aiming, roundtime reduction; AGI->Defensive Strength; AUR->Elemental CS + mana; WIS->Spiritual CS + spiritual mana; LOG->Mental CS + experience absorption + MTP; INT->experience/mental; DIS->field-exp pool size + mental/emotional resistance; INF->Bard/social CS. Stats grow only at level-up (increase when current level is divisible by floor(currentStat / SGI)); cap 100 raw.
  
_Build note:_ Pure computed function raw->bonus, then a coefficient table wires each stat into AS/DS/CS/TP/HP/exp. All coefficients live in ruleset data so a world can retune stat weightings.

**Character creation constraints and starting points**  
At creation the player distributes 440 attribute points across the 10 stats: minimum 20 in each; only FOUR stats may exceed 70; only ONE may exceed 90; max 100. After allocation the profession grants +10 to each of its two prime stats. Starting training points ~640, +10 per prime => ~660.
  
_Build note:_ Creation wizard validates the 440/20/four-over-70/one-over-90 rules and applies prime +10 from the Profession table; feeds directly into the TP and stat-bonus engines.

**Training points per level (PTP/MTP) and prime doubling**  
Per level: MTP = 25 + floor[(LOG+INT+WIS+INF+((AUR+DIS)/2))/20]; PTP = 25 + floor[(STR+CON+DEX+AGI+((AUR+DIS)/2))/20]. A profession's two prime requisites are DOUBLED inside these formulas. Practical max ~50 PTP / 50 MTP per level, rising to ~60 in a category when both primes fall there (e.g. Rogue with DEX+AGI primes can reach 60 PTP). After level 100 the rate becomes 1 PTP + 1 MTP per 2,500 experience (plus mastery/permanent TP).
  
_Build note:_ Computed engine recomputes PTP/MTP whenever level or a stat changes; prime-doubling reads primeStats from the Profession table.

**Full skill catalog, cost model, diminishing bonus, and rank caps**  
Skills by category — ARMOR: Armor Use (reduces attack RT/spell hindrance/maneuver hindrance; each +20 Armor Use bonus removes 1s of armor RT), Shield Use. WEAPONS: Edged, Blunt, Two-Handed, Polearm, Ranged (bows), Thrown, Brawling; plus Two Weapon Combat, Multi Opponent Combat, Ambush. COMBAT: Combat Maneuvers, Physical Fitness (cost 8/0; gives CMAN defense + health), Dodging. GENERAL: Survival, Disarming Traps, Picking Locks, Stalking and Hiding, Perception, Climbing, Swimming, First Aid, Trading, Pickpocketing. MAGIC: Arcane Symbols, Magic Item Use, Spell Aiming, Harness Power, Mana Control (Elemental/Spiritual/Mental), Spell Research (one per accessible circle), and the Lores — Elemental (Air/Earth/Fire/Water), Spiritual (Blessings/Religion/Summoning), Mental (Divination/Manipulation/Telepathy/Transference/Transformation). Every skill has a PHYSICAL/MENTAL cost (shown as e.g. First Aid '1/2') plus a parenthesized 'times per level'. Cost DOUBLES for each additional purchase within the same level. Bonus per rank has diminishing returns: +5/rank at ranks 1-10 stepping down to +1/rank at ranks 41+. At level 100 skills cap around 101 (1x), 202 (2x), or 303 (3x) ranks depending on how many times/level the profession may train them. (2021 change: CMAN/CML costs are now profession-agnostic.)
  
_Build note:_ THIS is the central ruleset dataset: Skill catalog {name, category, ptpCost, mtpCost}; a Profession x Skill 'maxTimesPerLevel' matrix (0x/1x/2x/3x); the diminishing-bonus bracket table; and skill->derived-stat wiring. Rank->bonus is a pure computed function; the per-level doubling-cost and cap are enforced at the training screen.

**Core combat resolution: AS vs DS + AvD + open d100 endroll**  
EndRoll = AttackStrength(AS) + AvD + d100 - DefensiveStrength(DS); the attack HITS if EndRoll > 100. AvD (Attack-versus-Defense) is a table value from weapon type x defender armor group. The d100 is OPEN-ENDED (~5% chance to roll <1 or >100). AS is built from weapon-skill bonus (scaled by stance) + STR/DEX bonus + spells + enhancives; DS from AGI + Dodging/Shield/Parry (stance-scaled) + spells + enhancives. Example from source: AS +109 vs DS +44 with AvD +39 + d100 +70 = +174 endroll (hit).
  
_Build note:_ The authoritative combat resolver lives on the Evennia server as a pure function reading AvD + DF tables from ruleset data; use a seeded RNG for reproducibility/replay (mirrors the netcode determinism discipline already in the multiplayer spec).

**Damage and critical-rank calculation (armor divisors + randomization)**  
Raw damage = (EndRoll - 100) x Weapon Damage Factor (DF). Critical Rank = truncate[ ((EndRoll-100)xDF - ArmorPaddingPoints + WeaponWeightingPoints) / ArmorCriticalDivisor ]. Armor critical divisors: robes/unarmored 5, leather 6, scale 7, chain 9, plate 11 (so 5-11 raw damage = a rank-1 crit vs unarmored). Critical randomization converts a computed 'max rank' into an actual rank uniformly between ceil(max/2) and max (table: rank1->1, rank2->1-2, rank3->2-3 ... rank9->5-9 max). Fatal thresholds vary by location x damage type (most head/neck/chest/abdomen kills need rank 6-8; head crush/impact fatal at rank 5; eye puncture fatal at rank 4). A crit sets stun duration, wound severity, bleeding, and extra damage.
  
_Build note:_ Critical tables are pure ruleset data keyed [damageType][bodyPart][rank] -> {woundRank, bleedRate, stunSec, extraDamage, fatal}. LLM generates the flavor message string per table cell; the engine only picks the cell.

**Hit locations, wounds/scars, HP, bleeding, death**  
Unaimed hit-location distribution (equal-height): arms 19.7%, legs 16.5%, chest 14.7%, abdomen 12.1%, back 10.1%, hands 9.0%, neck 7.4%, eyes 5.3%, head 5.2%. Wounds have three ranks per body part: R1 minor (mostly combat-maneuver penalties), R2 moderate (deep lacerations/fractures, usually BLEEDS, substantial penalties), R3 severe (severing, blindness, blocks casting/ranged/searching/sneaking depending on location). SCARS are partially-healed remnants (R1 scar = no mechanical penalty; R2 scar penalizes). Bleeding drains Health Points each ~7s round; HP pool scales with CON+level; reaching 0 HP or a lethal head/neck crit = death; the body then decays on the race timer.
  
_Build note:_ Server tick engine advances bleed/HP each round; per-body-part wound state machine feeds injury->penalty modifiers back into the combat and skill engines. Worldbuilder-authored 'you are here' wound diagram is a Godot client panel.

**Healing ecosystem (empath transfer, herbs, first aid, cleric)**  
EMPATH: TRANSFER a patient's wound onto themselves (leaves NO scar), then self-heal via the Empath Base (1100s) circle — able to heal every injury type by roughly level 18; Mental Lore Transference (~to 50 ranks) aids scar transfer, Transformation (~to 57) speeds self-heal; IMPRINT lets an empath remotely transfer wounds/scars (not blood loss) to imprinted patients. HERBS: eating a healing herb reduces a wound one level but leaves a scar; separate scar-removal herbs reduce scarring; herb-eating RT is reduced by First Aid at -1 RT per 20 skill bonus. FIRST AID TEND: bandaging temporarily stops bleeding; bandage duration lengthens and RT shortens with more First Aid. CLERIC/spiritual heals and Raise Dead round it out.
  
_Build note:_ Model healing as data-driven 'effects' (herb table entries, spell effects) that mutate the wound state machine; Empath/Cleric are first-class living-world NPC roles that can heal players using the same engine.

**Magic: three spheres and the spell-circle number map**  
Three spheres: Elemental, Spiritual, Mental. Spell circles and their numeric ranges: Minor Spiritual 100s (shared by ~8 professions), Major Spiritual 200s, Cleric Base 300s, Minor Elemental 400s, Major Elemental 500s, Ranger Base 600s, Sorcerer Base 700s, Wizard Base 900s, Bard Base 1000s, Empath Base 1100s, Minor Mental (~1200s), Paladin Base 1600s, plus Arcane (Sorcerer + shared arcane). A spell number = circle base + slot (e.g. 401, 425, 911). Pures learn 2 minor + their base circle (3 total); semis learn 1 minor + their base; squares up to 2 minor circles. Mana capacity grows via the Harness Power skill; more powerful spells cost more mana.
  
_Build note:_ Spell tables in ruleset {number, circle, sphere, type(warding/bolt/attack/utility/buff/heal), manaCost, castRT, effect, prepMsg}. Worldbuilder + LLM author flavor; computed engine handles cost/effect/hindrance.

**Casting mechanics: prep, cast RT, CS vs TD, bolts, armor hindrance**  
PREPARE is now instantaneous (verb is a legacy hold-over); actual cast roundtime is ~3 seconds; a prepared spell holds ~30s or until released on a valid target. WARDING spells resolve like combat: CS - TD + CvA + d100 > 100 to fully succeed (partial effects at lower margins). Casting Strength (CS) = caster level + Spell Research ranks + casting stat (AUR elemental / WIS spiritual / LOG mental). Target Defense (TD) = target level + defensive magics. CvA is casting-vs-armor. BOLT spells instead use Spell Aiming vs the target's DS (like a ranged attack). Heavier armor causes spell failure (hindrance), reduced by Armor Use.
  
_Build note:_ Two resolver paths on the server: warding (CS/TD/CvA/d100) and bolt (Spell Aiming/DS), both pure functions over ruleset data; armor-hindrance % is a computed value from armor group minus Armor Use.

**Spellburst (anti buff-stacking gate for high-level areas)**  
Spellburst penalizes a character wearing too many BENEFICIAL spells from OUTSIDE their profession's circles when inside flagged high-level hunting areas — inflicting hitpoint loss, critical damage, or even death. A character's safe-buff tolerance is raised by training the magic support skills: Magic Item Use, Arcane Symbols, Harness Power, and Spell Aiming.
  
_Build note:_ Computed 'burst tolerance' = f(MIU, Arcane Symbols, Harness Power, Spell Aiming) vs count/level of foreign active buffs; the server enforces it only in worldbuilder-tagged spellburst rooms.

**Combat Maneuver List (CMAN) — a large data-driven maneuver catalog**  
A character earns 1 CMAN point per rank of Combat Maneuvers; maneuvers are bought from the CML (mastering one typically costs ~20-30 points; since 2021 costs are the same for all professions). Categories and examples — ATTACK: Berserk, Mighty Blow, Coup de Grace, Spin Attack, Staggering Blow, True Strike, Leap Attack, Exsanguinate, Mug. SETUP: Feint (drop stance), Disarm Weapon, Sweep, Trip, Hamstring, Kneebash, Shield Bash, Sunder Shield, Cheapshots, Dirtkick, Eyepoke, Groin Kick, Headbutt, Throatchop, Tackle, Cutthroat, Subdue. PASSIVE/SPEC: Weapon/Parry/Evade/Block/Punch/Kick/Grapple Specialization, Combat Toughness/Mobility/Movement/Focus, Cunning Defense, Precision, Spell Parry. MARTIAL STANCES: Inner Harmony, Predator's Eye, Striking Asp, Whirling Dervish, Rolling Krynch, Duck and Weave, Executioner's Stance, Griffin's Voice, Slippery Mind, Flurry of Blows. BUFFS: Burst of Swiftness, Surge of Strength, Ki Focus, Internal Power, Bearhug, Garrote, Retreat. AoE: Bull Rush, Eviscerate. Shield-specific maneuvers key off Shield Use. Maneuvers consume STAMINA.
  
_Build note:_ CML is a textbook ruleset table {name, category, ranks, pointCost, staminaCost, RT, effect}; add a stamina resource engine. This is the single highest-leverage data-driven system to expose in the ruleset builder.

**Crafting/artisan trades + enchanting + enhancive items**  
Artisan cap 1200 ranks lets a character master 2 artisan skills; Forging is special (its ranks count 1/3 so a forger can master every weapon type). Trades: FORGING (a weapon = 2 parts, a blade/head + a hilt/haft; requires wearing an apron, a self-crafted forging-hammer, and a workshop/forge), ALCHEMY (a guild skill for Cleric/Empath/Sorcerer/Wizard — combine ingredients and process them with equipment to make items/potions), Outfitting/Tailoring, Engineering, Remedies (herbalism), Cobbling (footwear). ENCHANTING via Enchant (925): each success permanently adds +1 to a weapon/armor/shield bonus, paid in ESSENCE (wizards accrue essence = experience; caps 50,000/week and 200,000 total; F2P earns none). Essence cost is tiered (+0 to +24 ~= 312 x bonus level; +25 costs 7,500 then +7,500 per point above 24, e.g. +49->+50 = 195,000). Items must be pre-unlocked with colored ayan'eth (ROYGBIV) potions (green -> +20 in four +5 pours; blue -> +25; higher tiers beyond). Difficulty = trunc[((+Y - 2)^2)/9]. Success factors: Wizard Base ranks (+2/rank up to 1x level), character level, Magic Item Use/Arcane Symbols, LOG/INT, workshop (+25 public / +75 private), familiar. Failure loses 10% of the essence but never destroys the item. ENHANCIVE ITEMS grant stat/skill bonuses with limited charges.
  
_Build note:_ Crafting = recipe tables + a skill-check computed engine; the enchant essence-cost curve and difficulty formula are pure ruleset data; enhancive items are item-property modifiers that feed the stat/skill engines.

**Economy: coins, bank, gem/pawn shops, lockers/vaults, boxes/traps/locks**  
Currency is silver coins (the main unit; copper/bronze subunits exist); large sums convert to promissory NOTES for transport. BANKS in each town deposit/withdraw silvers and notes. GEM/PAWN shops buy gems and loot for silver (collectible shops don't buy). LOCKERS are rented storage — 50 item slots on standard subscription, 100 on premium — plus an account-wide FAMILY VAULT. TREASURE BOXES drop from creatures and are opened via a skill loop: DETECT the box -> if trapped, DISARM it (needs a lockpick) -> PICK it with a lockpick; governed by the Picking Locks and Disarming Traps skills (and Lock Mastery). Mishandling a trap triggers damaging effects.
  
_Build note:_ Coins/items persist in Evennia; bank/shop/locker are worldbuilder-authored room services; the box loop is a computed skill-check minigame (Picking Locks / Disarming Traps bonus vs box difficulty) that makes rogues mechanically distinct.

**Experience: field pool, absorption pulse, and the mind-state ladder (the core loop clock)**  
Earned experience first enters a FIELD (unprocessed) pool of size 800 + LOG + DIS (+ subscription: F2P 0 / Premium 200 / Platinum 400; + Wisdom-of-the-Ages 2/month). It ABSORBS into permanent 'mind' each pulse: on-node = 25 + LOG/5 + poolBonus + groupBonus + superNode(+2); in-town off-node = 22 + LOG/5 + ...; wilderness = 19 + LOG/7 + ...; poolBonus = +1 per 200 unabsorbed (cap +10 at 2000+); grouping with a non-F2P player = +1/pulse. Mind-state ladder by pool fullness: 'clear as a bell' (0%) -> 'fresh and clear' (>0%) -> 'clear' (>25%) -> 'muddled' (>50%) -> 'becoming numbed' (>62%) -> 'numbed' (>75%) -> 'must rest' (>90%) -> 'completely saturated' (>100%, no more field exp banks). Absorbed experience yields training points/levels; level cap is effectively 100, after which you gain mastery/permanent TP per 2,500 exp.
  
_Build note:_ A server 'pulse' scheduler drives absorption and the mind-state gate — this rhythm (hunt to fill the pool, rest/absorb, repeat) IS the GS4 gameplay loop and must be built early. 'Nodes' are worldbuilder-tagged rooms with an absorption multiplier.

**Death, deeds, resurrection, societies, subscription tiers**  
On death, unabsorbed field experience is LOST unless a Chrism is used; the corpse decays on the race timer; the character suffers Death's Sting (Constitution reduction, sharply reduced absorption). DEEDS are small purchased favors from Lorminstra (goddess of death/rebirth); one is auto-consumed per death to cushion the penalty. Resurrection: Cleric Raise Dead (318), Symbol/shrine revival. SOCIETIES are cross-profession and you may belong to only one at a time: Order of Voln (banishing undead; one new symbol per rank, 26 at Master), Guardians of Sunfist (anti-Grimswarm; HQ in Zul Logoth), Council of Light. Guilds (profession-specific) are separate from societies. Subscription tiers (F2P / Premium / Platinum) scale the exp pool, locker size, essence, and estate/estate-holder perks.
  
_Build note:_ Society membership = an unlockable ability track (rank -> granted symbol/ability), pure ruleset data; deeds are a death-insurance micro-economy; worldbuilder authors society HQs, shrines, and guild halls.

**Living world / autonomous agents (our differentiator over GS4)**  
GS4's world is persistent and hand-authored but its NPCs are largely static scripts. Our edge is simulated agent lives: town NPCs, shopkeepers, roaming adventurers, and society members with real personalities, psychology, daily schedules, relationships, and reactions to world events (e.g. a Grimswarm invasion, player reputation, festivals). Because the medium is text, the simulation can be arbitrarily deep.
  
_Build note:_ An NPC = a WORLDBUILDER entity (home/job/relationships/timeline) + a COMPANION-AI personality/psychology (consume-only) + an LLM voice (dialogue/rumors) + an Evennia daily-routine scheduler. Critically, NPCs are first-class actors that use the SAME computed combat/skill/magic engines as players, so an autonomous ranger hunts and an empath heals by the identical rules.

# PART 2 — Depth: Lifestyle, Trades, Magic & Premium (the 'live in the world' systems)

> Deep-dive research (workflow wf_45d8245a) into the non-combat GS4 depth Jesus flagged: trader/routes, crafting apprenticeship, moon/lunar magic, premium/estate. **Key corrections:** "Trader" and the "Moon Mage" are **DragonRealms** (GS4's sister game), not GS4 — GS4 has 10 combat-gated professions (Warrior/Rogue/Monk/Paladin/Ranger/Cleric/Empath/Sorcerer/Wizard/Bard) and FOUR moons (Liabo/Lornon/Tilaok/Makiri) with no lunar-magic class. ## GS4 Non-Combat Lifestyle — the TRADER profession & living economy (with DragonRealms as the mercantile-depth reference)

Key correction up front: "Trader" is NOT a GemStone IV profession. GS4 has exactly ten professions (Warrior, Rogue, Monk, Paladin, Ranger, Cleric, Empath, Sorcerer, Wizard, Bard) and its progression is COMBAT-GATED — you level by earning experience, and the deepest merchant/economy systems live in its sister game DragonRealms (DR). DR has a full Trader guild built around trade routes, caravans, commodity speculation, market tables and player shops, and it uses SKILL-BASED progression (the "Trading" skill circles you up), which is exactly the non-combat prosperity loop Jesus wants. So the build recipe is: take the LIVING-ECONOMY plumbing from GS4 (silvers, banks, lockers/Family Vaults, pawnshop/gemshop/furrier chain, Adventurer's Guild bounties, player-run shops, auctions, premium homes/estates) and graft the DR TRADER PROFESSION on top (routes, caravans, buy-low/sell-high commodity pits with supply-demand price bands, haggling via Charisma, the merchant guild, horse-brokering & hireling services). Everything below is labeled GS4 vs DR. Also correcting the brief's moon note: the three moons Katamba/Xibar/Yavash + moongates are DragonRealms (the Moon Mage); GS4 actually has FOUR moons (Liabo, Lornon, Tilaok, Makiri) and NO lunar-magic profession — both are captured as implementable targets. The economy is highly data-driven and maps cleanly onto a Ruleset builder: goods, markets, routes, price curves, and caravans are all just tables + a price formula.

**Trader is DR-only; GS4 profession landscape**  
GS4 has 10 professions and none is a merchant: Warrior/Rogue/Monk (combat), Paladin/Ranger (hybrid), Cleric/Empath/Sorcerer/Wizard/Bard (pure casters). GS4 'non-combat' is really non-combat SERVICES layered on a combat character (empaths heal-for-tips at town hubs, clerics bless/resurrect, bards do lore ID, warriors assess weapons / craft sheaths, rogues pick locks). DragonRealms, by contrast, has a dedicated TRADER guild whose whole identity is 'making money, pure and simple' via transport contracts, commodity speculation, and running market tables/shops. (gswiki.play.net/Category:Professions; elanthipedia.play.net/Trader)
  
_Build note:_ Model 'Trader' and 'Crafter' as first-class PROFESSIONS in the ruleset (DR-style), not just service verbs. Keep GS4's service-verb income as a secondary layer any profession can use.

**Progression model: combat-gated (GS4) vs skill-based (DR) — the crux of non-combat prosperity**  
GS4 leveling is fundamentally experience/combat-gated: there is no pure-trade path to level up. DR uses independent skill ranks — the exclusive 'Trading' skill earns experience from moving contracts, selling commodities, completing work orders, selling gem pouches/bundles, foraging/skinning tasks (~3 mindstates each), and pawn sales; a Trader 'circles' (advances) by training Trading + Appraisal. DR is the model that lets you PROSPER AND ADVANCE without combat. (elanthipedia.play.net/Trading_skill; Trader_new_player_guide)
  
_Build note:_ Decision to bake into the ruleset: give every economic action (sell, haggle, deliver contract, craft) an XP yield feeding a skill/circle track, so a player can level to endgame purely by trading. This is the single most important design choice for 'prosper without combat.'

**Commodity trading — the core buy-low/sell-high loop + PRICE FORMULA**  
DR commodity pits at Crossing, Arthe Dale, Leth Deriel. You read a commodity board; each good shows a SUPPLY STATE that sets price inversely to stock. Documented bands: Nearly Out (1-9 units → scarcity premium), Going Fast (10-49 → above avg), Good Stores (50-79 → mid), Surplus (80-100+ → discounted; an outpost in surplus REFUSES to buy more of that good). Each good has a low/high price band, e.g. Diamonds 1617-1991, Salt 27-51 silvers. Profit = buy where in surplus, sell where nearly-out at another pit. Charisma directly raises sell price (and thus Trading XP). (elanthipedia.play.net/Commodity_trading)
  
_Build note:_ Price formula (data-driven): price = lerp(good.high, good.low, stock/stock_max) * (1 + charismaBonus) * townRaceBias. Selling into a market RAISES its stock (price falls); buying LOWERS stock (price rises) — gives self-correcting supply/demand. Store per-good low/high bands + per-market stock as ruleset rows; a background tick can drift stock toward a regional baseline to regenerate arbitrage.

**Contract trading (the caravan delivery job)**  
DR contracts assign a specific good + destination outpost; you rent a caravan (initial rental 400 kronars at Zoluren outposts), load the crate, haul it along trade routes, and hand the contract to the destination's SHIPPING CLERK who pays out and unloads. Contracts EXPIRE if late; unfinished crates can be dumped at destination without pay. Lower reward than commodities but lower knowledge barrier. Guild NPCs: shipment clerks, commerce ministers, trade ministers coordinate movements. (elanthipedia.play.net/Trader; Trade_Route)
  
_Build note:_ Contract = generated quest row {good, originMarket, destMarket, payout, expiryTicks, penalty}. This is the beginner on-ramp before free-form commodity speculation. LLM/companion-AI can flavor each contract (who's shipping, why).

**Caravans & pack animals (the vehicle + mobile vault)**  
Caravans travel major trade routes and BYPASS normal road dangers/skill checks. Controlled via TELL <CARAVAN> TO: FOLLOW / LEAD TO <city> / GO TO / STOP / WAIT / RETURN / JOIN / SPOOK, plus speed GO SLOW/FAST. Constraint: routes cannot cross water needing ferry/boat. Circle gates: 35th = storage box (acts as a MOBILE VAULT) + exterior/plate customization; 50th = buy interior for 750 platinum + furniture. Dead/starved caravan → guild lecture + surcharge on future contracts. Pack animals (Aesry island) are the cheaper early alternative; must be fed (foraging feeds them free). Caravans stabled at 12 locations; reclaim fee grows after 4 andaen. (elanthipedia.play.net/Caravan)
  
_Build note:_ Caravan = entity with capacity, speed, storage, upkeep(feed), and a state machine (idle/leading/parked/dead). Model 'road danger bypass' as a risk-reduction stat so on-foot hauling stays riskier — creates the risk/reward Jesus wants.

**Carrying capacity & the numbers that gate a trade run**  
Max caravan commodity holdings = (CIRCLE × 4) + (2 × TRADING). This ties haul size to both level and the Trading skill, so bigger arbitrage runs are an earned reward. Trader skill weights (per DR): Trading & Appraisal are primary hard-requirements; Armor/Survival secondary; Weapon/Magic (Lunar mana) tertiary. Circle-200 cumulative targets: Trading 1,660 / Appraisal 1,450 / total Lore 6,400. (elanthipedia.play.net/Commodity_trading; Trader)
  
_Build note:_ Expose capacity as a ruleset formula referencing character stats so the Ruleset builder can tune it without code. Appraisal skill should gate how accurately a player SEES a good's true value vs. a noisy estimate — a fun information-asymmetry mechanic.

**Haggling / Charisma / Appraisal — the social-skill layer**  
Charisma is 'of utmost importance': it increases profit on every NPC sale, and more profit = more Trading XP. Appraisal reveals value (in GS4 the APPRAISE verb in a gemshop shows a gem's EXACT value). There is no spell that buffs Trading in DR, but GS4 Bards can raise gem sale value with Purification Song (1004). No explicit dice-off 'haggle mini-game' is documented — haggle is a stat modifier on price. (elanthipedia.play.net/Trader; gswiki.play.net/Gemshop)
  
_Build note:_ Implement haggling as a Charisma-vs-merchant-disposition check returning a price multiplier, optionally an interactive back-and-forth (LLM-driven dialogue that resolves to the multiplier). Appraisal skill sets the confidence interval on displayed prices.

**Trade-route risk, theft & justice mechanics**  
Thieves can steal from active trade routes. Trader guild abilities: 'Trade Route Justice' (accuse a thief caught on an active route → guild dispatches enforcers to punish), 'Caravan Spook' (force your caravan to drag you out of combat, minimal damage), 'Corpse Hauling' (tie a dead PC to the caravan to move them out of restricted areas). (elanthipedia.play.net/Trader)
  
_Build note:_ Risk = per-route ambush/theft probability (weighted by cargo value, escort presence, time of day/season). Escorts, guards-for-hire, and caravan upgrades reduce it. This is the natural place to intersect the combat and trade loops for players who want both.

**GS4 coin economy: silvers, promissory notes, banks**  
Silver coin is GS4's common currency. Every town has a BANK for deposit/withdraw; large sums convert to PROMISSORY NOTES for safe transport. Multiple coin denominations exist across the world (kronars are DR's currency; GS4 uses silvers). Towns also carry weapon shop, armor shop, pawnshop, furrier, consignment, gemshop. (gswiki.play.net/Silver_coin; Bank)
  
_Build note:_ Currency + Bank are ruleset primitives: account balance, per-town cash, promissory-note items (bearer instruments that can be stolen/traded). Support multi-currency if we want regional coin (kronars vs silvers) as another arbitrage axis.

**Lockers, Family Vaults & storage economy**  
GS4 basic accounts: 1 locker holding 50 items in your creation town. Premium: lockers in EVERY town holding +50 (up to 80) items, transferable between towns almost instantly. Family Vault = account-wide shared locker accessible by all characters on the account. Storage overflow and item weight/count limits push players toward banks/lockers and create demand for premium storage. (gswiki.play.net/Locker)
  
_Build note:_ Locker = per-(account,town) container with item cap; Vault = per-account shared container. Instant inter-town transfer for premium is a clean paywall/reward toggle in the ruleset.

**Gemshop / pawnshop / furrier valuation chain**  
Gems & precious-metal items (non-combat gear) fetch ~3x more at the GEMSHOP than the pawnshop. Gemshops cap single-item payout ~25,000-50,000 silvers and may refuse very valuable/magical/enhancive pieces. Sell many gems at once via SELL <container>; jewelry sells individually. Prices vary by town AND character race (bias 0 to +5). Furrier buys skins/pelts; consignment shops handle resale. Traders (DR) get a ~60% profit bonus selling gems/furs. (gswiki.play.net/Gemshop; elanthipedia.play.net/Trader)
  
_Build note:_ Each shop type = a buyer with {acceptedCategories, payoutMultiplier, singleItemCap, refusalRules}. Town+race bias is a small lookup table. Gems become the universal liquid loot → coin pipeline, and a Trader's bonus is a profession perk row.

**Adventurer's Guild bounties — non-combat & combat income + a parallel currency**  
GS4 Adventurer's Guild issues level-scaled bounties: creature kills (combat), heirloom retrieval, SKIN collecting (→ furrier), GEM collecting (obtain 5-10 of a named gem, sell to jeweler; flat 1000 XP + variable bounty points/silver), HERB foraging (must be fresh), ESCORT travelers between towns, and RESCUE lost children — several are fully non-combat. Pay is silver + BOUNTY POINTS, a separate currency spent only at the Guild's treasure master (e.g., Crystal Amulet 100 pts, Gold Ring 500, Random Magic Item 1000, Society Reset 500,000, Potion of Body & Mind 1,000,000). (gswiki.play.net/Adventurer's_Guild; Bounty_point)
  
_Build note:_ Bounty board = generator of typed tasks with {type, target, qty, levelBand, silverReward, bountyPoints}. Ship non-combat task types (escort/herb/gem/skin) so a pacifist can grind them. Bounty points = second-currency store — great sink and progression alt-track.

**Player-run shops, market tables & auctions (player-driven commerce)**  
GS4 PLAYER-RUN SHOPS (since 2004): owner buys furniture/sign/door/decor from a furniture merchant; inventory limited by counter/room — 25 distinct items per room (stacks up to 50 each), up to 3 rooms (only 1 room on non-premium) = max 75 items. Shops are searchable and purchasable remotely by shop number. DR MARKET TABLES: 20th-circle Traders rent a table for 4 RL hours / 2,500 kronars (20 items; re-rent after 1 RL hr); price tags 625 kronars/sack, item retrieval 3,125, currency conversion ~2 gold 25 coppers/item; abandoned goods held 24 RL hrs. DR player SHOPS at 30th circle (Crossing/Riverhaven/Shard plazas); AUCTION halls at 30th circle (Crossing/Riverhaven/Ratha). GS4 auctions run at festivals (Ebon Gate) and standalone great auctions, GM- or player-run. (gswiki.play.net/Player_run_shop; elanthipedia.play.net/Market_tables; Auction)
  
_Build note:_ Shop = ownable location with slotted display containers, a sign, an owner-set price per item, and a 'browse/buy while owner offline' flow (async player-to-player sales). Market table = the cheap timed entry tier; full shop = the endgame tier. Auction house = timed-bid entity. All map to ruleset rows + a small order-matching service.

**Premium / estate / player-driven systems (the play.net/gs4/premium page)**  
GS4 Premium tier: after 90 CONSECUTIVE days you may OWN A PRIVATE HOME in a player neighborhood — decorate with shop/merchant/raffle furniture, can include an interactive PET that follows you in the neighborhood; one character owns per account but all characters access it. Premium also grants 15 extra character slots, +50 locker capacity on every slot with a locker in every town (instant transfer), and 100 PREMIUM POINTS/month redeemable for items/upgrades. (Note: play.net/gs4/premium returned a 500 at fetch time; details confirmed via gswiki.play.net/Premium_subscription, Premium_home, Long-term_Benefits.)
  
_Build note:_ Homes/estates = ownable, decoratable interior locations gated by a loyalty/tenure counter; furniture is a catalog of placeable items (some cash/points-only). Premium points = a monthly-drip premium currency with its own store. Interactive pet = a lightweight companion-AI hook. Estates are the marquee non-combat 'lifestyle' sink.

**The moons & moongates — CORRECTION + implementable lunar systems (labeled)**  
Correction to the brief: Katamba/Xibar/Yavash and the Moon Mage + moongates are DRAGONREALMS. GS4 actually has FOUR moons — Liabo (largest; silver-white by night, golden by day; home of race-favoring 'good' Arkati; symbols are lucky charms), Lornon (the 'dark' Arkati), Tilaok, and Makiri — but GS4 has NO lunar-magic profession or moongate travel. In DR, the Moon Mage casts MOONGATE toward a chosen moon (Xibar=blue, Yavash=red, Katamba=black) after placing a 'focused moonbeam' anchor at the destination; mana scales with distance (5 same-zone → 13+ island-to-mainland, up to ~45 streams); casting on a moon that is NOT currently visible spawns a 'voidspawn' or instantly obliterates the caster; the gate is CYCLIC (needs continuous mana to sustain). Naming crossover: Liabo (GS4) = Katamba (DR). (gswiki.play.net/Liabo; drwiki.play.net/Moongate; elanthipedia.play.net/Moon_Mage)
  
_Build note:_ Capture BOTH: (a) GS4 four-moon SKY MODEL — moon phases/visibility drive flavor, festival timing, and could modulate magic or spawn tables; (b) DR MOONGATE as a data-driven fast-travel network — nodes = 'moonbeam anchors', edges gated by which moon is visible now, mana cost by distance, failure = spawn/penalty. Moon visibility is a global clock the worldbuilder computes; ties travel to time-of-night = emergent risk/reward and pairs with trade routes (lunar shortcuts vs road caravans).

### Data model (Ruleset-builder tables)

- Good {id, name, category(gem|ore|food|textile|spice|crafted|commodity), baseLow, baseHigh, weight, stackable, perishable, unitLabel} — the tradeable item type; low/high define the price band (e.g. Salt 27-51, Diamond 1617-1991).
- Market {id, townId, type(commodity_pit|gemshop|pawnshop|furrier|consignment|bank|auction_house|player_shop|market_table), acceptedCategories[], payoutMultiplier, singleItemCap, refusalRules} — a place that buys/sells; town+race bias table (0..+5) applied on top.
- MarketStock {marketId, goodId, stock, stockMax, supplyState(nearly_out 1-9 | going_fast 10-49 | good_stores 50-79 | surplus 80-100+), lastDriftTick} — per-market inventory that drives price and regenerates over time toward a regional baseline.
- PriceFormula: price(marketId, goodId, actor) = lerp(good.high, good.low, stock/stockMax) * (1 + charismaBonus(actor) + traderProfessionBonus(actor)) * townRaceBias(town, actor.race); selling raises stock (price drops), buying lowers stock (price rises); surplus markets refuse further buys.
- Route {id, name, originTownId, destTownId, waypoints[], distance, waterCrossing(bool), baseTravelTicks, ambushRisk, allowedForCaravan(bool)} — connects markets; caravans may only use routes with no ferry/boat water crossing.
- Caravan {id, ownerId, capacity=(circle*4)+(2*trading), speed, storageBoxSlots(unlocked@circle35), interiorTier(@circle50), upkeepFeedLevel, riskReduction, state(idle|leading|parked|dead)} — the hauling vehicle / mobile vault; commands FOLLOW/LEAD TO/GO TO/STOP/WAIT/RETURN/SPOOK.
- PackAnimal {id, ownerId, capacity, feedLevel, forageable(bool)} — cheaper early-game alternative to a caravan.
- Contract {id, goodId, qty, originMarketId, destMarketId, payout, expiryTick, latePenalty, issuerNpc} — generated delivery job; completed by handing to destination shipping clerk.
- Bounty {id, type(creature|heirloom|skin|gem|herb|escort|rescue), target, qty, levelBand, silverReward, bountyPoints, isCombat(bool)} — Adventurer's Guild task; non-combat types (escort/herb/gem/skin) enable pacifist income.
- Currency & Wallet {silvers, kronars(optional regional), promissoryNotes[] (bearer items), premiumPoints, bountyPoints} — multi-currency purse; Bank {townId} holds deposits, issues promissory notes.
- StorageContainer {kind(locker|family_vault|shop_display|caravan_box), scope(town|account), itemCap, contents[]} — locker per-(account,town), Family Vault per-account shared, premium raises caps + enables instant inter-town transfer.
- Shop {id, ownerId, townId, rooms(<=3, <=1 non-premium), displaySlotsPerRoom=25, stackMax=50, sign, furniture[], listings:[{goodId, price, qty}]} — ownable player storefront with async browse/buy-while-offline; MarketTable is the timed cheap-tier variant (20 items, 4hr rental).
- AuctionLot {id, shopOrHallId, goodId, startBid, currentBid, highBidderId, closeTick} — timed player/GM auction.
- Estate/Home {id, ownerAccountId, neighborhoodId, tenureUnlock(90 days), furniture[], interactivePet(companionAiRef)} — premium lifestyle property; one owner per account, all chars access.
- SkillTrack {characterId, skill(Trading|Appraisal|Forging|Fletching|Cobbling|Tailoring|Alchemy|Riding), rank, xp} + XpYield on each economic action so trading/crafting alone can circle a character up (DR-style non-combat progression).
- WorkOrder {id, tradeSkill, requiredGoodId, qty, difficulty, coinReward, xpReward, issuerNpc} — standing crafting demand (DR Crafting Society) that links crafters into the same market economy.
- MoonSky {moons:[Liabo,Lornon,Tilaok,Makiri (GS4) | Katamba,Xibar,Yavash (DR)], phase, visibleNow(bool)} — global clock; Moongate {anchorNodeId, moon, manaCost=f(distance 5..45), sustainDrain, failureEffect(voidspawn|obliterate) when moon not visible} — data-driven lunar fast-travel network (DR), labeled distinct from GS4's four-moon flavor sky.

## GS4 Crafting, Apprenticeship &amp; Player-Run Establishments (with DragonRealms depth harvested, labeled)

Key reframe for the build bible: GemStone IV's OWN crafting is deliberately thin — three "Artisan skills" (Fletching, Cobbling, Forging) that give minor stat edges over shop gear, plus magical crafting (Alchemy, Enchanting) locked to caster professions. The DEEP, career-defining crafting-and-apprenticeship economy Jesus described (blacksmith/tailor/alchemist trades, apprentice-to-master grind, running trade ROUTES as a merchant class) lives in Simutronics' SISTER game DragonRealms, not GS4. Because we're text and unconstrained, the right target is a HYBRID: adopt GS4's rank ladder + forging fidelity + player-shop/premium-estate systems verbatim, and graft DragonRealms' five-Lore-skill x three-discipline tree, Crafting Society apprenticeship (logbooks, tiered recipe books apprentice->journeyman->master, work orders, prestige, maker marks), Trader trade-route/caravan/commodity economy, and full gather->refine->craft supply chain on top. Every mechanic below is data-drivable: recipes, materials, tools, workshops, ranks, and trainer/shop NPCs are all authorable in the Ruleset builder + Worldbuilder, with companion-AI NPCs as the masters/shopkeepers/innkeepers. Two clean differentiators where BOTH source games are thin: true player-run INNS/TAVERNS/SMITHIES (GS4 only has generic player "Trade Shops" + NPC-run inns), and a unified cross-discipline economy loop. Sources are Play.net official + GSWiki + Elanthipedia (all first-party Simutronics wikis).

**GS4's native crafting is intentionally SHALLOW (the reframe)**  
GemStone IV's own player crafting is the 'Artisan skill' system with only THREE implemented skills: Fletching (arrows/crossbow bolts/thrown darts), Cobbling (shoes/boots/sandals/slippers, pockets at high rank), and Forging (weapons only). Mining and Smelting are listed as PLANNED. Crafted items give only a MINOR stat edge over normal shop gear. Ranks: Novice 0-99, Apprentice 100-199, Journeyman 200-299, Skilled 300-399, Highly-Skilled 400-499, Master 500. Cap 1,200 artisan ranks total = master two skills + dabble a third; forging ranks count as 1/3 so you can master crafting + all five weapon types. Commands ARTISAN SKILLS / ARTISAN UNLEARN. Free-to-play accounts have NO artisan access.
  
_Build note:_ Adopt the rank ladder and cap math verbatim, but treat GS4's 3-skill roster as a floor — the depth Jesus wants comes from grafting DragonRealms' tree on top.

**GS4 Forging is a deep, multi-station simulation (our flagship physical craft)**  
Best-in-class fidelity to copy: weapons are two parts (head/blade + handle/haft). Tools: crafting apron, glyphs (patterns), grinder, forge, mithril tongs, polisher, vise, slab-cutter, forging hammer (attune if made by another). Pipeline: slab-cutter (MEASURE/POKE/PUSH/PULL to cut to weight) -> grinder (TURN, Crafting skill) -> forge (GET TONGS with hammer in right hand; per-metal cycle counts) -> tempering trough (oil chosen by enchant tier) -> polisher (LEAN) -> vise (TURN to assemble). Quality ladder Perfect->Superior->Elegant->Fine->Nice->Plain->Simple->Crude->Flimsy with concrete deltas (Perfect = +2/+2 STR/DU, +6% dmg, +3 AvD; Flimsy = -10/-10, -6%, -3). Quality from Extraordinary-Success rolls (grinder 5%, forge 5-8% w/ perfect hammer, vise up to 10%). Success = roll vs skill + averaged stat bonuses - material plus/weight penalties + hammer mod - profession handicap + racial bonus (Dwarf best at +50/+7).
  
_Build note:_ This whole pipeline is data-drivable as Recipe.steps[] with per-station skill + stat checks. Use it as the reference template for all physical crafts (smithing/tailoring/carving).

**Materials, tools & workshops (GS4 forging as the pattern)**  
Materials tier out: woods (Haon, Hickory, Ko'nag, Maoral, Maple, Modwir, Monir, Oak, Walnut); non-magical metals (bronze/iron/steel/invar — iron forged only in Kobold Mines, 604s cycle); common-magical (mithril/ora/imflass/vultite); uncommon (drakar/gornar/mithglin/rhimar/zorchar); rare (eahnor/eonake/faenor/golvern/kelyn/razern/rolaren/vaalorn/veil-iron/white+black ora). Tempering OILS set enchant plus: water (0x, free) -> tempering oil (0x, ~10s) -> enchanted (1-2x, ~2000s) -> twice-enchanted (3-4x, ~10k) -> ensorcelled (4x+, ~20k). Workshops: public workshops in every town via 'GO WORKSHOP' at supply shops, 1,000 silver/hour, group-leader-billed (whole group crafts).
  
_Build note:_ Model Material.enchant_plus + forge_cycle_time + rarity, and Workshop as public-rental vs player-owned. Rare-metal + high-oil combos are the endgame crafting economy sink.

**GS4 Alchemy & Enchanting are CASTER-gated magical crafts, not open trades**  
Alchemy is limited to Cleric/Empath/Sorcerer/Wizard and driven by GUILD RANK (not artisan ranks). Tools: cauldron, mortar & pestle, alembic (distill), crucible (extract), glass vials/crystal flasks (rank 15+). Products: general recipes (all-profession), profession-specific potions, and trinkets. Subskills unlock by rank: Seal/Boil/Simmer/Examine/Chant @1, Distill/Extract @15, Infuse @30, Refract @45, Channel @50; SEAL is always the final stabilizing step. Enchanting in GS4 is SPELL-based, not a bench trade: Wizard Enchant(925) gathers/spends 'essence' for a permanent +1 AS or DS per success; Create Magic Item(420) imbeds spells into imbeddables; Cleric Bless(304); Bards literally SING weapons/armor/shields into existence.
  
_Build note:_ Keep two enchant paths in the data model: spell-driven (GS4) AND bench-craft Artificing (DR). Alchemy shows how to gate a craft behind class + a separate guild-rank track.

**DragonRealms is the DEEP crafting model to harvest (labeled: sister game, not GS4)**  
DR structures crafting as 5 Lore skills x 3 disciplines: Forging{Blacksmithing (tools/smelting/containers/sundries), Armorsmithing, Weaponsmithing}; Engineering{Carving (bone/stone), Shaping (wood+clay), Tinkering (crossbows/bolts/instruments/lockpicks/siege/traps)}; Outfitting{Tailoring (leather+cloth), Jewelry, Artistry (embellishment/alterations)}; Alchemy{Remedies (potions/salves/tinctures), Reactants (poisons), Cooking (food/drink/stat buffs)}; Enchanting{Artificing (founts/foci/runestones/wands), Binding (constellation jewelry), Invoking (automatic devices)}. Quality runs 0 'practically worthless' -> 12 'masterfully-crafted'; 12 difficulty tiers; material workability applies a 40-300% difficulty multiplier; a 7-level 'understanding' scale gates reading recipes.
  
_Build note:_ This is the target breadth. Note DR labels: some disciplines (Cooking, Jewelry, Poisons, Binding, Invoking) are 'not yet released' in DR itself — we can implement them fully since we're unconstrained.

**The APPRENTICESHIP model — Crafting Societies, tiered books, work orders, prestige**  
DR's society system is exactly the apprentice->master arc Jesus wants. Society halls (e.g. Forging Society, Outfitting Society) contain a bookstore, equipment room (ORDER tools), supplies room, and work rooms. You 'apprentice' by buying tiered recipe books: Apprentice book (cheapest, trains to ~200 ranks) -> Journeyman book (~15 platinum, to ~700 ranks) -> Master book (700+). WORK ORDERS: buy a crafting logbook and RUB to bond it, then ASK <master> FOR EASY|CHALLENGING|HARD <discipline> WORK; completing orders pays coin + discipline PRESTIGE (+ Trading XP for Traders). Enough prestige unlocks MAKER MARKS (sign your goods). Specialization: ASK MASTER ABOUT CHOOSING CAREER/HOBBY <discipline> — career = larger bonus + up to 12 technique slots, hobby = small bonus + up to 6; general slots span a group (13). Slots unlock at ranks 25/50/75/100/150/200/250/300/400/500/700/900/1200. Choices are permanent barring a 100-plat yearly reset. CRAFT ASSIST lets up to 3 helpers boost a crafter.
  
_Build note:_ Map society-master NPCs to companion-AI 'masters'; logbook + work-order loop is the core apprentice grind; maker-marks give master crafters brand identity in the economy.

**Trader profession = the trade-ROUTE / merchant playstyle Jesus named (DR, labeled)**  
DR's Trader is a full non-combat merchant class ('what traders do best is make money'). Core skills Trading + Appraisal. Systems: (1) COMMODITY PITS in Crossing/Arthe Dale/Leth Deriel — buy low, sell high across cities on fluctuating prices; (2) CARAVANS/pack animals (rent ~400) for delivery CONTRACTS from outposts to destinations (non-cancelable, expire, surrender w/o pay); caravans lead you along FIXED ROUTES and keep moving during roundtime so you train while hauling; (3) TRADE-ROUTE JUSTICE — if robbed on a route, run to the nearest outpost and accuse the thief to summon guild enforcers; (4) GEM POUCHES / FUR BUNDLES with up to 60% bulk profit scaling on CHA*(Appraisal+Trading); (5) progression from rental market tables (20th circle) -> owned plaza shops (30th) -> AUCTION HALLS; (6) inter-city Shardstar SHIPPING. Traders also get bonus Blacksmithing + Artistry technique slots and access Lunar magic via 'The Negotiants'.
  
_Build note:_ Directly satisfies 'be a TRADER running trade ROUTES.' Model TradeRoute + Contract + CommodityPit + caravan; the 'train-while-travelling' route mechanic is a great text-MUD hook.

**Gather -> refine -> craft SUPPLY CHAIN (DR)**  
Gathering skills: Mining (rock/metal ore), Lumberjacking (wood), Foraging/Outdoorsmanship (healing herbs, wax for candles, sticks/limbs for Shaping, tea leaves, coffee beans, alchemical catalysts, grass/vines for braiding, branches for carving), and Skinning. Refinement chains: ore -> smelt -> ingot; component metals -> crucible -> ALLOY (alloys aren't mined, only crucible-made or looted); animal hide -> skin -> scrape -> cure -> leather; fibers -> spin -> thread -> weave -> cloth. Every craft draws from these upstream nodes, so gathering and crafting professions interlock into one economy.
  
_Build note:_ Model GatheringNode (region, yields, depletion) + Material.refine_chain. This is what makes a self-sustaining player economy — gatherers feed smiths/tailors/alchemists who feed traders.

**Player-run SHOPS exist in GS4 (Trade Shops) — verbatim system**  
A Trade Shop is a PC-owned business that (unlike fixed-inventory NPC shops) can sell ANYTHING. Acquire via a DEED from the Shop Permit Authority at town hall (town CITIZENSHIP required); 100k/150k/200k silver for 1/2/3 rooms. Basic sub = max 1 room, Premium = 3 rooms; monthly rent 10k (basic) / 30k (premium) silver. Full SHOP command suite: CLAIM, EXTEND/DOWNSIZE, SELL, PRICE, INVENTORY, BROWSE, INSPECT, PURCHASE (remote, via paid runner), BAN (up to 15), HAGGLE (race+profession discount bias), SIGN, ARRANGE, ROOF, PARTNER (up to 4, separate funds, can DEPOSIT), TRANSFER, LOTTERY, WITHDRAW, DEPOSIT, DIRECTORY, SALES (last 10). 3% sales tax (capped 1M/item). 25 different items/room, stacks of 50 (75 items in a 3-room shop). One shop per account. Repossessed after 3 months unpaid rent or owner absent 180 days (partner can take over at 90).
  
_Build note:_ Copy the SHOP verb suite and deed/rent/tax economy directly. Remote SHOP BROWSE/PURCHASE across towns = a built-in player marketplace index worth surfacing in the client UI.

**Player-run INNS/SMITHIES are a GAP in both games — our differentiator**  
Neither GS4 nor DR lets players truly run an INN/TAVERN or a public SMITHY-as-business. GS4 Trade Shops are generic goods stores (you can cosmetically theme one as a 'tavern' but there's no lodging/meal/room-rental service layer); inns and taverns are NPC-run. DR smithies are society/public workshops, not player-owned businesses with staff and apprentices. There is no first-class 'apprentice under a player-master at their shop' loop.
  
_Build note:_ Ship an Establishment type system (shop|smithy|inn|tavern|apothecary|bakery|guildhall) with companion-AI staff, service menus (lodging/meals/repair/craft-to-order), and a PLAYER APPRENTICE roster training under a player- or NPC-master. This directly delivers Jesus's 'run a shop or inn / apprentice under a smith' vision and exceeds both source games.

**Premium ESTATES & player organizations (GS4 lifestyle depth)**  
Premium homes: after 90 CONSECUTIVE premium days, claim a private home in a player NEIGHBORHOOD; homes range shack->standalone mansion; decorate with shop/merchant/raffle furniture ('billions' of styles), add an interactive follower PET, a scent, a guest list, and toggle familiar/locate access. One decorated home per account; all characters can enter. Player organizations: Co-operative Houses of Elanthia (CHE, the 'Great Houses') are large manor houses charging initiation fees + monthly dues, granting expanded lockers + private areas; you belong to only one CHE at a time. Meeting Hall Organizations (MHOs) are fully player-created/run orgs with open membership (you can join many).
  
_Build note:_ These map to PlayerEstate + PlayerOrganization entities. The estate furniture catalog and neighborhoods are pure Worldbuilder-authored data; CHE/MHO give the player-driven social/estate layer Jesus flagged from the premium page.

**GS4 vs DragonRealms labeling for the build bible**  
Explicit source separation so the team doesn't conflate them: GS4 (Elanthia) HAS — Artisan skills (Fletching/Cobbling/Forging) with the Novice->Master ladder, spell-based Enchant/Alchemy for casters, player Trade Shops, premium homes, CHE/MHO, and the three moons (Katamba/Xibar/Yavash) + moongates. The 'Moon Mage' class is DragonRealms (not GS4).
  
_Build note:_ When authoring, tag each mechanic GS4|DR|OURS so parity claims stay honest. Our template = GS4 shell + DR depth + our player-establishment/apprentice extensions.

**Spell numbering scheme & the three spheres**  
GS4 organizes all magic into three spheres — Elemental (natural conflict), Spiritual (deities/spirits), Mental (innate psychic). Every spell has a number = circle prefix * 100 + rank within the circle. Minor Shock = 901 (Wizard circle '9', 1st spell), so circles are colloquially named by their hundreds ('the 900s'). Each rank of skill trained in a circle unlocks the next-numbered spell up to the character's level.
  
_Build note:_ Model Spell.number as the primary key; derive circle and rank arithmetically. This single convention makes the whole spellbook data-drivable in the Ruleset builder.

**Complete circle-to-number map (16 circles)**  
Minor Spiritual 100s, Major Spiritual 200s, Cleric Base 300s, Minor Elemental 400s, Major Elemental 500s, Ranger Base 600s, Sorcerer Base 700s (hybrid elem+spir), Wizard Base 900s, Bard Base 1000s (hybrid ment+elem), Empath Base 1100s (hybrid spir+ment, mechanically spiritual), Minor Mental 1200s, Major Mental 1300s (designed, largely unreleased), Savant Base 1400s (unimplemented profession), Paladin Base 1600s, Arcane 1700s (item/scroll/creature spells, no profession). There is no active 800s or 1500s circle; GS4 Monks are non-casters.
  
_Build note:_ Seed the circle registry with exactly these number-series + sphere + access flags; keep 'released' booleans so unimplemented circles (1300/1400) exist as data without gameplay.

**Shared vs profession circles + the two-slot foreign-spell rule**  
Minor/Major Elemental, Minor/Major Spiritual, and Minor/Major Mental are the shared 'open' circles a profession draws from based on its sphere access; each profession also gets a Base circle. Wizards access 3 circles (Minor+Major Elemental+Wizard); Monks access 2. Characters can only wear a limited number of beneficial spells from OUTSIDE their profession's circles before penalties.
  
_Build note:_ Represent access as SpellCircle.access + Profession.circleAccess[]; the foreign-buff limit is the input to the spellburst model below.

**Casting pipeline (PREPARE -> activate, INCANT, mana)**  
Two paths: (1) PREPARE <number|mnemonic|name> readies a spell for 30s, then activate with CAST / CHANNEL / EVOKE / SUMMON; (2) INCANT does prep+activation in one command. Bards SING their songs. Mana capacity grows via the Harness Power skill; more powerful spells cost more mana, and attempting a spell without enough mana risks injury or death (no safe fizzle).
  
_Build note:_ Model activationVerb per spell and a mana pool sized by Harness Power. Enforce insufficient-mana as a damage event, honoring the no-fallback rule.

**Three combat-resolution paths + Casting Strength**  
Bolt spells use an attack roll (Attack Strength vs Defense Strength) and need aim/line of sight. Warding spells breach magical defense via Casting Strength (CS) vs the target's Target Defense (TD). Maneuver spells force a physical reaction roll (Spell Maneuver Roll). CS and TD are first-class derived stats.
  
_Build note:_ Give Spell.type a resolver enum (bolt/warding/maneuver) that routes to AS-vs-DS, CS-vs-TD, or SMR. This is the core of combat parity for magic.

**Armor hindrance + spell failure**  
Heavier armor interferes with casting and can make spells fail; the chance is reduced by training the Armor Use skill. This is how GS4 gates casters out of heavy plate without hard class locks.
  
_Build note:_ Data table: armorGroup -> baseSpellFailure%, reduced by Armor Use ranks. Pure casters effectively train little Armor Use.

**Spellburst — the foreign-buff penalty engine**  
A character can only 'wear' so much beneficial spell-mana from spells outside their profession's circles; exceeding the limit causes HP loss, critical damage, or death. Rules: learnable spells' mana is HALVED for the calc; GROUP and ARCANE spells do NOT count; mass spells DO count; offensive/active-list spells don't count; spells with spellburst mana 9+ (Strength, Mass Blur, Mobility, Bravery, Heroism) are extra volatile. Training Arcane Symbols and Magic Item Use raises tolerance and can negate it entirely.
  
_Build note:_ Implement as: sum(foreignBuff.spellburstMana with halving/exemptions) vs tolerance(Arcane Symbols, MIU). This is a distinctive, very implementable text-game system that rewards a build's own-circle buffs.

**[DR-inspired] Moon Mage guild — the deep moon magic to adapt**

**[DR-inspired] Focus Moonbeam + Moongate spell**

Astral Travel uses enchanted 'Grazhir' shards as anchors for long-distance/rogue teleport.
  
_Build note:_ Anchor-then-consume pattern (place beam -> spells reference it) is elegant and text-native. Color-code beams to moons; require outdoors + moon visibility; make Moongate a cyclic mana-streamed portal.

## GS4 Premium / Estate / Player-Driven Endgame — the "live in the world" lifestyle layer (research for the text-MUD/MMORPG template; Godot client + Evennia server; parity target GemStone IV / Elanthia)

GemStone IV's non-combat endgame is a deep, persistent "live in the world" layer that we can implement fully because it's text. It rests on a three-tier subscription (Basic $14.95 / Premium $39.95 / Platinum $49.95 monthly) — there is NO separate "Estate Holder" tier; home ownership is a Premium perk gated behind 90 consecutive days of membership (one home per account). Premium grants ~100 loyalty "Premium Points"/month (a spendable currency for alterations, titles, verbs, enchants), 15 extra character slots, per-town lockers, ~10 transports/month, a monthly WPS chance and Spell-of-the-Month, and access to Premium-only zones (Four Winds Isle → Mist Harbor town + Four Winds Halls lounges). The housing system is remarkably deep for text: homes are classified by Location(urban/rural) x Class(lower/middle/upper) x Structure(freestanding/door), the room description is ASSEMBLED from ~10 furniture slots (floor, walls, bed, table, fireplace, door, window, chairs, rug, wall-hanging) via HOME CHANGE, and homes move between towns via a Cartage Merchant + promissory-note + deed ritual. Private Property (PP, up to 10 rooms) and the Bungalow System (middle-class, 3 rooms) extend this with workshops, stills, shrines, extra rooms, storage nodes and transport keys — sold at Grand Auctions, raffles, and festival "Property Troves." Player-run shops (deed from a permit office in 9 towns, citizenship required, 10k silver/room rent, 3% sales tax, 25 item-types/room, remote SHOP BROWSE/PURCHASE) create a real player economy. The merchant/festival economy (Ebon Gate = death-themed, annual, running since 1998; Duskruin = biannual bloodriot arena + Bloodscrip; Rumor Woods) runs on paid SimuCoin passes and event currencies (soul shards, bloodscrip, seashells) and is where the high-end alteration/enchant/auction services and the Gilded Locus enhancive live. The social/roleplay endgame layers on top: three Societies (Order of Voln, Council of Light, Guardians of Sunfist) with rank tasks, profession Guilds joinable at L15 (guild skills, dues, Guild Master mentorship), a rich Title system (level/experience/skill/Adventurer's-Guild titles via TITLE LIST), Deeds from the goddess Lorminstra, and Artisan trade skills (Forging, Fletching, Cobbling — 500 ranks each) that feed the trader/crafter dream. All of this maps cleanly onto our stack: ruleset builder holds the tier/currency/rent/festival-cadence config; Worldbuilder authors neighborhoods, property/shop rooms, furniture catalogs and festival zones; the companion-AI library drives realtors, cartage merchants, guild masters and background shop customers as NPC lives; LLM generation assembles furniture-slot room descriptions and festival/title flavor; Evennia persists deeds/ownership/inventory and hosts the HOME/SHOP verb commands.

**Subscription tiers — correcting the 'Estate Holder' assumption**  
GS4 has THREE subscription tiers, not an 'Estate Holder' tier: Basic $14.95/mo, Premium $39.95/mo (includes Basic), Platinum $49.95/mo (includes Premium). 'Estate Holder' is not a real GS4 subscription level — home/estate ownership is instead a PERK of Premium, gated behind 90 consecutive days of Premium membership, one home per account, and requires keeping the subscription active to retain the home. Platinum adds: bonus enhancive location slots (ring/neck/wrist/earring/earrings), +400 field experience pool, and +750 SimuCoins/month (vs +250 at Premium).
  
_Build note:_ Model tiers as ruleset-builder config (data-driven), NOT hardcoded. Because we control the ruleset we CAN promote GS4's implicit 'premium homeowner' into an explicit first-class 'Estate Holder' tier if desired — flag this as a deliberate parity-plus decision, labeled as our extension.

**Premium Points as a monthly loyalty currency ('Long-term Benefits')**  
Premium members receive ~100 'Premium Points' per month for loyalty; they accumulate and are spent at the Lighthouse on Four Winds Isle. Sample costs: alterations 300–600 pts, titles 600 pts, extra typeahead line 900 pts, enhancive items 1,800–2,400 pts, custom verbs/titles 2,500–5,500 pts, enchanting/flares/padding 100–25,000+ pts, weekly profession resource points 25 pts (once/week). Notably there are NO tenure/milestone gates ('play X months → reward') — loyalty is converted directly to spendable purchasing power.
  
_Build note:_ Perfect fit for our Currency model + a 'Lighthouse' vendor authored in Worldbuilder. Ruleset defines the monthly grant and the point-cost table; this is the backbone of non-combat progression. We could optionally ADD milestone/veteran rewards (which GS4 lacks) as a parity-plus.

**Premium-only zones: Four Winds Isle, Mist Harbor, Four Winds Halls**  
Premium unlocks Four Winds Isle (reached via transport jewelry), which hosts Mist Harbor — a full Premium-only town with all normal town services/shops. 'Four Winds Halls' are premium lounges in multiple towns offering refreshments, nodes, transport jewelry, premium flowers, Black Drake gambling tables, and interactive room props. The annual Premium Festival is also held on Four Winds Isle.
  
_Build note:_ Worldbuilder authors these as access-gated zones; Evennia enforces the tier gate on room/exit traversal. Interactive props + gambling tables are great LLM-flavor + minigame surface. Zone-access is a clean data flag (zone.requiredTier).

**Premium utility perks: lockers, transport, WPS, Spell-of-the-Month, extra typeahead, Reim**  
Premium grants: a locker in EVERY town holding +50 items each, transferable between towns nearly instantly; the Premium Transport System usable up to ~10x/month; a monthly chance to win WPS (Weighting/Padding/Sighting) item services; a Premium Spell-of-the-Month (usable 1–2x daily, rotates monthly); an extra type-ahead command line (prevents queue overflow); Reim settlement access on a 22-hour cooldown without buying quest tokens; and faster staff response on assists.
  
_Build note:_ Each is a small data-driven entitlement toggle on PremiumTier. Lockers = per-town persistent storage container (Evennia object owned by account). Transport = rate-limited teleport with monthly counter. WPS/Spell-of-Month = monthly rotating reward table in ruleset.

**Premium Homes — eligibility, acquisition, and tier taxonomy**  
After 90 consecutive Premium days you buy a home at the town realtor (variable fee, e.g. ~100,000 silver) then go to a neighborhood room and type HOME CLAIM. Homes are classified on three axes: Location (Urban/Rural) x Class (Lower/Middle/Upper) x Structure (Freestanding or Door-based), plus optional race/class restrictions. Flavor by tier: Lower Rural = hut/shack/sagging wooden door; Upper Urban = 'residence' or sturdy metal door. The original type PERMANENTLY determines the exterior appearance. ONE home per account (all account characters can access it, but only the owning character decorates).
  
_Build note:_ Home tier is a pure data enum (location x class x structure). Exterior is derived + locked at claim time. HOME CLAIM maps to an Evennia command that assigns an authored neighborhood room to a character. Realtor = companion-AI NPC.

**Home room descriptions are ASSEMBLED from furniture slots (the killer text mechanic)**  
Each home has ~10 furniture categories: Floor (required), floor covering/rug (optional), Walls (required), wall hanging (optional), Large Item #1 (typically bed), Large Item #2 (typically table/desk), fireplace/heat source, Door (can be hidden), window (optional/hidden), chairs (optional). Buying a new item in a category REPLACES the old one. The LOOK room description is generated by assembling filled slots via HOME CHANGE (preset configurations). Verbs: HOME CHANGE (layout), HOME SWAP (swap bed/table positions), HOME SCENT, HOME SAVE (persist desc/scent/privacy/guest).
  
_Build note:_ This is the single best fit for LLM generation in the whole spec: define a furniture catalog (Worldbuilder-authored items, each with a descriptive fragment), and have the LLM (or a template engine for determinism) compose the room prose from filled slots. Store the assembled description + slot state in Evennia. Fully implementable and deep because it's text.

**Home mobility, deeds, selling — the Cartage Merchant ritual**  
HOME DEED CONFIRM produces a deed and makes the home 'deserted' for 96 hours (only the deed-holder can re-claim in that window). To MOVE: hold deed in right hand + promissory note in left hand at a Cartage Merchant (e.g. Wehnimer's Ceraleth's Cartage in Moot Hall) to ship furnishings, then HOME CLAIM the new home with the SAME deed (buying a new deed breaks the transfer). To SELL: sell to town clerk for fair price (home+furnishings), or give the deed to another player (they claim within 96h). Other verbs: HOME GUEST, HOME PRIVACY {FAMILIAR|LOCATE}, HOME TRANSFER.
  
_Build note:_ Deed = a first-class transferable object with a state machine (owned → deserted(96h) → claimed / inTransit). Cartage Merchant = companion-AI NPC service that migrates the furniture-slot state between authored rooms. Great emergent player-economy surface (deed resale).

**Private Property (PP) vs Bungalow System — the higher home tiers**  
Beyond basic Premium Homes there are deeded Private Properties: the Bungalow System (BS, 'middle class', max 3 rooms, released at the Grand Auction of 5123, cannot access limited-release Property Trove offerings) and full Private Properties (PP, max 10 rooms, usually sold with 4–6, eligible for BOTH off-the-shelf and limited-release upgrades). One deeded property per account. Town properties may require citizenship (e.g. an Aies apartment requires Ta'Illistim citizenship). Properties use special keys (rogues with lock mastery can duplicate). Monthly rent may apply; non-payment → repossession.
  
_Build note:_ PP/Bungalow are just Estate/Home records with higher roomCount and an upgrades list. Citizenship gate = data flag checked by Evennia. Multi-room properties = linked authored rooms. Key-duplication is a fun rogue-skill interaction to preserve.

**Property upgrades catalog (workshops, stills, shrines, storage, transport)**  
Unlimited (off-the-shelf) upgrades: traskcans, storage nodes, transport keys, keyless entry, storage expansion, MOARdrobes, foraged props, insects, window scripts, permanence certificates. Limited (event Property Trove) upgrades: stills, mailboxes, additional rooms, workshops, shrines, and property renaming. Upgrade shops include 'The Roof Over' and 'Vintage Visions'.
  
_Build note:_ Upgrades = an entitlement list on the Estate/Home record, each unlocking a feature (workshop→enables crafting in home, still→brewing, mailbox→messaging, transport key→fast travel node). This is where the persistent-home layer connects to the crafting/trader lane (workshop = home crafting station).

**Player-run shops — ownership, rent, tax, inventory**  
Buy a shop deed from a permit-office NPC in 9 towns (Teras, Wehnimer's, River's Rest, Icemule, Solhaven, Zul Logoth, Ta'Illistim, Ta'Vaalor, Mist Harbor). Requires full/partial citizenship + Premium (3-room shops) or Basic (1 room). One shop/account. Rent = 10,000 silver/room/month auto-drawn from shop balance; 3% sales tax auto-deducted per sale (capped 1,000,000 silver/item). Each room holds ≤25 item TYPES, items stack ≤50 (loresinged can't stack). Stock via SHOP SELL, price via SHOP PRICE. Miss 3 months rent → repossession (inventory crated 90 days at permit office).
  
_Build note:_ Directly realizes Jesus's 'run a shop' goal. Shop = persistent Evennia object with an inventory table + a balance ledger; rent/tax are ruleset-config formulas run on a monthly tick (Evennia scripts/cron). Permit office clerk = companion-AI NPC.

**Shops are a real economy: remote browse/purchase, partners, haggle, bans**  
Customers browse distant shops with SHOP BROWSE [TOWN] [SHOP#], inspect via SHOP INSPECT, buy remotely with SHOP PURCHASE (+5,000 silver courier fee, or a Locker Runner Contract). Owners run haggle discounts by race/profession (SHOP HAGGLE), take up to 4 partners (separate inventories/profits — owner can't touch partner profits), and ban up to 15 players. If the owner doesn't visit for 90 days a partner can ASSIST to claim it; if unvisited 180 days the owner loses it. Verbs: SHOP CLAIM/PARTNER/TRANSFER/INVENTORY/DIRECTORY/EXTEND PERMIT/ARRANGE/DOWNSIZE.
  
_Build note:_ Remote SHOP BROWSE across all towns = a global marketplace query over the shop DB (Evennia). Partners + per-partner profit ledgers = multi-owner permissions. Haggle rules = data. This is the player-driven commerce backbone; background NPC shoppers (companion-AI) can generate passive sales so a shop feels alive when the owner is offline.

**Festival/merchant economy — the beating heart of high-end item services**  
Three seasonal paid merchant festivals drive the endgame economy: EBON GATE (annual, late Oct/early Nov, death/Eve-of-Reunion theme, running since 1998 — 'longest-running regularly scheduled merchant event', venue rotates yearly e.g. Evermore Hollow/Caligos Isle, ~1 month long); DUSKRUIN (biannual, Bloodriven Village, a bloodriot combat-arena + sewers + heist event); RUMOR WOODS (nature/Elanthian). Entry via SimuCoin passes (e.g. Ebon Gate Merchant Pass = 3,000 SimuCoins; games 10–100 SC/entry). Each has its own currency: Ebon Gate = soul shards (Evermore) or seashells (Caligos); Duskruin = Bloodscrip (SCRIP command; Arena/Sewers ≈300 each/run + random gear).
  
_Build note:_ Festivals = time-boxed Worldbuilder-authored zones with their own currency wallets and vendor tables, toggled on by a ruleset calendar (Evennia cron). The paid-pass model can become a soft-currency/entitlement gate rather than real-money. Duskruin's Endless Arena (leaderboard + titles) is a strong reusable minigame.

**Festival high-end services: alterations, dabbler, Gilded Locus, troves, live auctions**  
Festivals are where premium item services live: feature ALTERATIONS (e.g. 200 soul shards/full-feature-line at Ebon Gate; Spidersilk Apothecary alter-box); the DABBLER (automated NPC that STRIPS properties — weighting, padding, sanctification, defender, sighting, flares, scripts — for a fee); enhancive swapping (NPC Sylinar); automated flaring services; the GILDED LOCUS (account-attuned enhancive updatable ONLY at Duskruin/Ebon Gate/Rumor Woods, non-transferable); the PROPERTY TROVE (buy homes/private properties + limited upgrades at events); Treasure Trove raffles; and live MANIA events with auctions, reverse auctions, and spins where players pre-deposit currency to bid.
  
_Build note:_ Alteration/dabbler/enchant = item-service NPCs that mutate item property records — huge for text (item descriptions are just data we rewrite via LLM). The Property Trove ties festivals into the home system (buy/upgrade estates at events). Auctions = a bidding subsystem worth building once and reusing (Grand Auction, festival mania, player-to-player).

**SimuCoin store + Grand Auction — real-money & rare-item pipeline**  
SimuCoins (SC) are the real-money add-on currency across all Simutronics games; buy on play.net, redeem in-game with SIMUCOIN DELIVER. They buy festival passes, custom properties, cosmetics, and store items. The Grand Auction is a marquee live event selling the rarest items (some from HESS, some found nowhere else) and is how new property systems debuted (Bungalow System launched at the Grand Auction of 5123).
  
_Build note:_ Keep the SimuCoin layer as an abstract 'premium currency' in our Currency model — we can drive it via subscription grants rather than real money for a template. The Grand Auction is a scheduled, high-stakes reuse of the auction subsystem; a natural sink for elite player wealth.

**Social/roleplay endgame — Societies (Voln / Council of Light / Sunfist)**  
Three Societies grant abilities + roleplay identity, joined one-at-a-time: ORDER OF VOLN (anti-undead; 26 master symbols, a new usable symbol per rank); COUNCIL OF LIGHT (rank advancement via 4 NPCs — Master of Gifts, Master of Service, Inquisitor, Taskmaster; skinning-collection tasks); GUARDIANS OF SUNFIST (Adventurer's-Guild-style task system with milestone tasks at ranks 5/10/15/20). Each offers unique combat/utility bonuses AND roleplay hooks.
  
_Build note:_ Societies = a membership record + rank + task-ledger + ability unlocks; rank NPCs and task-givers are companion-AI NPCs. Data-driven ability grants. Strong non-combat identity/progression axis that our ruleset builder can template as reusable 'faction' schemas.

**Professional Guilds (level 15), Titles, Deeds, and Artisan trade skills**  
Profession GUILDS are joinable at level 15 (auto-invite): guild skills supplement training via ranked task-based progression (Warrior/Rogue guilds are complete; others partial), members pay initiation + monthly dues (500–15,000 silver), and Guild Masters (rank 125 + 1 mastered skill) can mentor/collect-dues/promote for exp+fame. TITLES come from level, experience, skill, and Adventurer's-Guild task counts (TITLE LIST PRENAME/POSTNAME; failures subtract from task totals). DEEDS are granted by the goddess Lorminstra for donation/sacrifice (in-game puzzles; 10 for premium, 5 F2P via Gift of Lorminstra). ARTISAN trade skills — Forging (6 sub-skills, sought-after weapons that stack with enchant/bless), Fletching (arrows/bolts/darts), Cobbling (footwear), planned Mining/Smelting — each has 500 ranks; Alchemy is a guild skill. F2P is excluded from artisan skills.
  
_Build note:_ This is the crafter/trader backbone Jesus wants: Artisan skills = 500-rank ladders producing goods that STOCK player shops and TRADE across towns. Guild Master mentorship + dues = social/economic loop. Titles = the visible lifestyle 'score'. All data-driven: skill ladders, title tables, and deed puzzles author cleanly in Worldbuilder + ruleset.

**GS4 vs DragonRealms — labeled cool targets for BOTH (moons / moongates)**  
Clarifying the sister-game distinction: the 'MOON MAGE' profession (moon-based lunar magic, astrology/prediction, telepathy) is DRAGONREALMS, not GS4. GS4 instead has the three moons Katamba (black), Xibar (silver/grey), and Yavash (small pink) whose phases modulate certain magics, plus MOONGATES (Cleric/portal-style teleportation tied to the moons/planes). Both are implementable and cool: capture DR's Moon Mage divination/lunar-lore as a full lunar caster archetype AND GS4's three-moon phase system + moongate travel network.
  
_Build note:_ OUT of the premium/estate focus but flagged per the brief.
